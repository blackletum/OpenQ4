# Phase G recon — fog + blend lights (GL ground truth → vk port surface)

## 1. Backend draw path (all in src/renderer/draw_common.cpp)

**RB_STD_FogAllLights (7762-7816)** — called from RB_STD_DrawView tail at 9802-9809, after RB_STD_ForceAmbient (9798) and RB_STD_LightGridIndirect (9790), before post-fog material passes (9817-9819); gated by ModernGL pass-ownership skip (9803). Early-out: `r_skipFogLights || r_showOverDraw != 0 || viewDef->isXraySubview` (7765-7767). Disables GL stencil test for the whole pass (7773) and re-enables at exit (7815) — a D3XP-disabled stencil anti-double-fog guard sits in `#if 0` (7782-7805). Walks viewLights, skipping lights that are neither `IsFogLight()` nor `IsBlendLight()` (7778); fog → `RB_FogPass(vLight->globalInteractions, vLight->localInteractions)` (7807-7808), blend → `RB_BlendLight(same two lists)` (7809-7810). **translucentInteractions are never fogged/blended.**

**RB_FogPass (7633-7754)**:
- Cap surface: local drawSurf_t with `space = &backEnd.viewDef->worldSpace`, `geo = vLight->frustumTris`, `scissorRect = backEnd.viewDef->scissor` (full view scissor, NOT the light scissor) (7649-7652); skipped if `!frustumTris->ambientCache` (7646).
- Fog color/density from light shader **stage 0 only** (7658): `lightColor[0..3] = regs[stage->color.registers[0..3]]` (7660-7663), `glColor3fv` (RGB only; alpha defaults 1) (7665).
- `a = RB_FogDistanceScale(lightColor[3])` (7668) — draw_common.cpp:240-246: alpha ≤ 1.0 → `-0.5/DEFAULT_FOG_DISTANCE` (500.0f, tr_local.h:57), else `-0.5/alpha`.
- State: `GLS_DEPTHMASK | GLS_SRCBLEND_SRC_ALPHA | GLS_DSTBLEND_ONE_MINUS_SRC_ALPHA | GLS_DEPTHFUNC_EQUAL` (7670). openQ4 hardening: force ARB programs off (7675-7679), disable color arrays (7686-7689).
- Texture 0 = `globalImages->fogImage` (_fog), GL_MODULATE, S/T texgen, Q pinned via glTexCoord2f(0.5,0.5) (7682-7694). Planes (idPlane fogTexGenPlanes[4], draw_common.cpp:232, enum tr_local.h:63-70):
  - `FOG_DISTANCE_PLANE_S = a * worldSpace.modelViewMatrix[2,6,10,14]` (view-space Z = eye depth) (7696-7699)
  - `FOG_DISTANCE_PLANE_T = a * modelViewMatrix[0,4,8,12]` (computed but overridden per-surface to constant 0.5) (7701-7704)
- Texture 1 = `globalImages->fogEnterImage` (_fogEnter), GL_MODULATE, S/T texgen (7708-7715):
  - `FOG_ENTER_PLANE_T = 0.001f * vLight->fogPlane` (7718-7721)
  - `s = vieworg · enterT.Normal() + enterT[3]` (7724-7725); `FOG_ENTER_PLANE_S = (0,0,0, FOG_ENTER + s)` — constant S = viewer's scaled distance to fog plane (7727-7730). `FOG_ENTER = (64+1)/(64*2) = 0.5078125` (tr_local.h:59-60).
- Draw order: global then local interaction chains via `RB_RenderDrawSurfChainWithFunction(..., RB_T_BasicFog)` (7736-7737); then re-state `GLS_DEPTHFUNC_LESS` (→ glDepthFunc(GL_LEQUAL), tr_backend.cpp:289-296) + `GL_Cull(CT_BACK_SIDED)` and draw frustumTris (7741-7744), restore CT_FRONT_SIDED.

**RB_T_BasicFog (7593-7624)** — per-space plane math on space change: `R_GlobalPlaneToLocal(surf->space->modelMatrix, plane, local)` for each; T0.S gets `local[3] += 0.5` (7599-7601); T0.T forced to constant `(0,0,0,0.5)` (7605-7606); T1.T gets `local[3] += FOG_ENTER` (7611-7613); T1.S transformed unmodified (7615-7616; zero-normal plane stays constant). Then MD5R packed surfaces take `RB_ARB2_MD5R_DrawBasicFog` (7619-7620) else `RB_T_RenderTriangleSurface` (tr_render.cpp:247-256, position-only). The ARB vprog twin (draw_arb2.cpp:1551-1593, env params 93-96 = FOG_DISTANCE_PLANE / DISTANCE_BIAS(0,0,0,0.5) / ENTER_PLANE_T / ENTER_PLANE_S, enum draw_arb2.cpp:77-87) is the exact vertex-shader contract to port.

**RB_BlendLight (7508-7582)** — skip if no surfs or `r_skipBlendLights` (7514-7519). Texture 1 = `vLight->falloffImage`, S texgen only, constant T=0.5 via glTexCoord2f(0,0.5) (7526-7530). Texture 0 = per-stage `stage->texture.image` with S/T/Q texgen (7532-7537,7550). Walks **all** light-shader stages (unlike fog's stage 0): condition-register gate (7543); `GL_State(GLS_DEPTHMASK | stage->drawStateBits | GLS_DEPTHFUNC_EQUAL)` — blend factors come from the light material stage's blend keyword (7547); optional stage texture matrix via RB_LoadShaderTextureMatrix (7552-7554, GL texture-matrix based — the vk equivalent is folding it into the texgen planes with `RB_BakeTextureMatrixIntoTexgen`, already ported at vk_Interactions.cpp:137-172); `lightColor RGBA = regs[stage->color.registers[0..3]]`, glColor4fv (7557-7561); draws both chains per stage (7563-7564). Cull is left front-sided.

**RB_T_BlendLight (7462-7497)** — on space change: `lightProject[i] = R_GlobalPlaneToLocal(surf->space->modelMatrix, backEnd.vLight->lightProject[i])` for i=0..3 (7470-7476); tex0 S/T/Q object planes = lightProject[0..2], tex1 S = lightProject[3] (falloff) (7478-7484). Vertex pointer = xyz from ambientCache (idDrawVert stride) with a shadowCache_t fallback (7488-7494) — edge case for shadow-only geo. Fragment result (fixed function) = projected texture (S/Q, T/Q) × falloff(S, 0.5) × stage color, blended by drawStateBits.

## 2. Surface lists + fogPlane/viewHeight math

- Fog/blend draw over the light's **interaction lists** (globalInteractions then localInteractions). For fog/blend lights, interaction creation uses `R_SurfaceShaderCreatesLightTris` → `surfaceShader->ReceivesFog()` instead of ReceivesLighting (Interaction.cpp:52-60); `ReceivesFog = IsDrawn() && !noFog && coverage != MC_TRANSLUCENT` (Material.h:548-549; `noFog` keyword Material.cpp:2855-2857). List routing at Interaction.cpp:1966-1975 (MC_TRANSLUCENT → translucentInteractions [never fogged]; noSelfShadow+shadows → localInteractions; else globalInteractions) via R_LinkLightSurf (tr_light.cpp:1098-1149: frame-allocs drawSurf with geo/space/material/lightScissor + shaderRegisters; r_limitBatchSize can drop surfs at 1101-1105).
- **frustumTris**: `vLight->frustumTris = light->frustumTris` and `vLight->fogPlane = light->frustum[5]` copied at tr_light.cpp:937-938 (viewLight_t fields tr_local.h:219-220). The front-end vertex-caches frustumTris **for fog lights only** at tr_light.cpp:1680-1691 (`R_CreateAmbientCache(light->frustumTris, false)` + touch; skip light if out of memory) — idDrawVert data in the standard vertexCache, indexes heap-resident on the lightDef.
- fogPlane is global-space; backend scales by 0.001 (1 texel ≈ 1000 units) before the FOG_ENTER offset. "viewHeight" is baked into the **_fogEnter** lookup: `FogFraction(viewHeight, targetHeight)` (Image_intrinsic.cpp:281-342; RAMP_RANGE=8, DEEP_RANGE=-30, heights < 0 are inside fog), indexed by (S = viewer distance-to-plane, T = fragment distance-to-plane), both scaled+offset as above; 64×64 (`FOG_ENTER_SIZE`, tr_local.h:59), TF_LINEAR/TR_CLAMP, RGB=white + alpha=fraction, RGBA classification is deliberate (Image_intrinsic.cpp:352-380).
- **_fog** image (Image_intrinsic.cpp:228-271): 128×128 radial ramp `alpha = 255*(1 - 0.982^(255*d))`, edge texels forced 255 (258-260), TR_CLAMP, RGBA-white. Sampled at (S = 0.5 + a·eyeZ, T = 0.5) → 1D density ramp by eye depth. Images registered as `_fog`/`_fogEnter` (Image_intrinsic.cpp:687-688).

## 3. Light-stage walk for blend lights

Covered in RB_BlendLight above: every stage of the light material (conditionRegister-gated), per-stage GLS blend bits, per-stage RGBA color from registers, texture from `stage->texture.image` projected by lightProject[0..2] (with optional per-stage texture matrix), falloff via lightProject[3] + falloffImage. `vLight->lightProject[0..3]` / `falloffImage` copied from lightDef at tr_light.cpp:933-939. Light shader registers evaluated per view at tr_light.cpp:1587-1591; note the zero-light early-skip explicitly exempts fog/blend lights (tr_light.cpp:1595).

## 4. What the vk implementation needs

- **Insertion point**: VK_GuiExecutor_Draw3DView already splits ambient walks into pre-fog/post-fog passes (vk_GuiExecutor.cpp:1802-1885, pass 0 = decals + sort<SS_MEDIUM, pass 1 = SS_MEDIUM..<SS_POST_PROCESS, comment 1825-1834). The fog/blend pass must run **between pass 0 and pass 1** to mirror RB_STD_DrawView (draw_common.cpp:9774 pre-fog / 9806 fog / 9818 post-fog). GL order overall: depth fill → interactions (VK_Interactions_DrawLights, called at 1796) → pre-fog ambient → fog/blend → post-fog ambient.
- **Existing conventions suffice**: per-space MVP `VK_BuildSurfMVP` (vk_GuiExecutor.cpp:1183-1195, depth hacks + clip-z fixup); `VK_Exec_SetSurfScissor` (1152-1178) handles both the per-surf light scissor and the frustumTris' viewDef->scissor; `VK_Exec_BindTriGeometry` (1098-1143) — **frustumTris flow through this same CPU-backed path unchanged**: it reads `vertexCache.Position(tri->ambientCache)` (which the fog-light front-end guarantees, tr_light.cpp:1682-1691) + CPU `tri->indexes`, streaming into the per-frame-slot vertex/index rings with split vert/idx memos; dynamic uniform ring set 6 with 256B slices (`VK_Exec_InteractionUniformAlloc`, 1232-1237) if push space runs out; pipeline cache keyed on GLS blend bits with dynamic depth/cull/bias (386-397; blend translation 307-321; dynamic-state list 332-342). Stencil is off in every pipeline — matching GL's explicit stencil disable around the fog pass.
- **Shader pairs (2 new)**:
  1. *Fog*: vertex evaluates 3 local planes (distanceS [+0.5 folded], enterT [+FOG_ENTER folded], enterS) → texcoords (S0, 0.5) and (S1, T1); exact contract = the MD5R fog vprog (draw_arb2.cpp:1573-1588). Fragment: `frag = vec4(fogColor.rgb, texFog.a * texEnter.a)`, blend SRC_ALPHA/ONE_MINUS_SRC_ALPHA, no depth write. Per-draw data = 3 planes + color = 4 vec4 → **fits the shared 128B push block** ({mat4; vec4 a,b,c,d}, vk_Interactions.cpp:77-86) with no uniform-ring traffic.
  2. *Blend light*: vertex outputs projective STQ from lightProject[0..2] + falloff S from lightProject[3] — a strict subset of interaction.vert's vLightProjectionTexCoord/vLightFalloffTexCoord math (Vulkan/shaders/interaction.vert:78-83). Fragment: `textureProj(lightImage, stq) * texture(falloff, vec2(s, 0.5)) * stageColor`. Per-draw data = 4 planes + color = 5 vec4 — one over the push block's spare space; either pack color into unused push floats or use a small set-6 ring block per the interaction convention. Needs 2 sampler sets (projection, falloff) — the cached per-image single-sampler sets (`VK_Exec_ImageDescriptor`, vk_GuiExecutor.cpp:1216-1224) already cover this.
- **Per-draw state**: depth test on / write off; compare EQUAL for surface chains; LEQUAL + GL back-sided cull (VK front-cull with the mirror swap, pattern at vk_Interactions.cpp:448-458) for the frustumTris cap; blend-light pipelines requested per stage `drawStateBits` from the existing blend-keyed cache. Fog needs one fixed SRC_ALPHA/ONE_MINUS blend pipeline over the new shader pair.
- **Light loop skeleton**: mirror RB_STD_FogAllLights — reuse the existing loop shape of VK_Interactions_DrawLights (vk_Interactions.cpp:613-722), which currently `continue`s past fog (683) and blend (686) lights ("do fogging later").
- **Gap note**: packed MD5R surfaces (`tri->primBatchMesh`) have zero vk handling (no primBatchMesh refs under src/renderer/Vulkan); the GL fog path materializes their caches via RB_EnsurePackedClassicDrawCaches (draw_common.cpp:5355-5381) / RB_ARB2_MD5R_DrawBasicFog — on vk such surfaces will be skipped exactly as the existing depth/ambient walks skip them (speculative impact: MD5R-heavy scenes under-fog until a packed-mesh story exists).
- shadowCache-fallback vertex binding (draw_common.cpp:7491-7494) has no vk analog (BindTriGeometry is idDrawVert-only); in practice fog/blend chains carry ambient-cached light tris — mark as accepted edge case.

## 5. Cvars + openQ4 extensions

- `r_skipFogLights` "0" CVAR_RENDERER|CVAR_BOOL (RenderSystem_init.cpp:493); `r_skipBlendLights` "0" (RenderSystem_init.cpp:492). `r_showOverDraw != 0` also disables fog (draw_common.cpp:7765). **No r_showFog exists.** Scene-packet capture honors both skips at capture time (ScenePackets.cpp:1450-1464).
- openQ4 deltas vs retail Doom3/Q4 fog:
  - Quake 4 `noFog`/ReceivesFog interaction gate (Interaction.cpp:52-60, 1566) and `MF_NOPORTALFOG` (tr_lightrun.cpp:321).
  - MD5R packed fog vprog path (draw_arb2.cpp:1551, params 93-96; batch variant RB_ARB2_DrawPackedMD5RFogBatches at 1556).
  - Fixed-function hardening in RB_FogPass: programs forced off (7675-7679), color arrays disabled (7686-7689), RGBA (not alpha-only) fog lookups (Image_intrinsic.cpp:268-270, 376-379).
  - Pre-fog/post-fog material-pass split with feedback-capture hooks around the fog pass (draw_common.cpp:9760-9819) — already mirrored by the vk ambient two-pass split.
  - ModernGL executor can own RENDER_PASS_FOG_BLEND ("modern-forward-fog-complete", ModernGLExecutor.cpp:6119-6120; fog draws route to the forward+ transparent shader, ModernGLDrawPlan.cpp:233-237) — a parallel programmable implementation, not a direct porting reference (clustered pipeline).
  - RENDER_PASS_FOG_BLEND scene-packet pass captures global+local chains per fog/blend light with the ReceivesFog + !SuppressInSubview filter (ScenePackets.cpp:1540-1557, filter 1319-1324) — the vk module consumes viewDef directly today, but this filter is the canonical eligibility statement.
  - Fogged-portal culling shares `a = -0.5f/DEFAULT_FOG_DISTANCE` in the front-end (RenderWorld_portals.cpp:149, R_CreateLightDefFogPortals tr_lightrun.cpp:310-323) — no backend work needed.
