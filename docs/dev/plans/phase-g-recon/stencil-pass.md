  - `r_useExternalShadows==0` → full `tri->numIndexes`;
  - `==2` → force `numShadowIndexesNoCaps` (testing);

  - else full indexes, internal.
  - If numIndexes==tri->numIndexes force `external=false` (7266-7268) — the front end forced NoCaps=NoFrontCaps=numIndexes for perforated/translucent casters and view-suppressed surfaces (Interaction.cpp:1641-1647), so they always take the robust path.
- Per-surface depth bounds: `glDepthBoundsEXT(surf->scissorRect.zmin, zmax)` (7271-7273); zmin/zmax are GL window-z from R_TransformEyeZToWin (tr_main.cpp:161-164).
- **Stencil op sequences** (ops are (fail, zfail, zpass)):
  - Two-sided single pass (7313-7340): `frontSidedFace = isMirror ? GL_FRONT : GL_BACK` (7321) — the face rasterized by legacy CT_FRONT_SIDED draws, because GL_Cull(CT_FRONT_SIDED) culls **GL_FRONT** in non-mirror views (tr_backend.cpp:193-221). GL_Cull(CT_TWO_SIDED); if `!external` first the z-fail "preload" ("patent-free work around"): `StencilOpSeparate(frontSided, KEEP, DECR_WRAP, DECR_WRAP)` + `(backSided, KEEP, INCR_WRAP, INCR_WRAP)` + draw (7330-7332); then z-pass: `(frontSided, KEEP, KEEP, INCR_WRAP)` + `(backSided, KEEP, KEEP, DECR_WRAP)` + draw (7336-7338). Net across both draws = classic z-fail (Carmack's reverse) without the patented single-draw depth-fail formulation.
  - Two-pass fallback (7342-7361): same math with cull flips — preload `glStencilOp(KEEP,decr,decr)` CT_FRONT_SIDED then `(KEEP,incr,incr)` CT_BACK_SIDED (only `!external`), then z-pass `(KEEP,KEEP,incr)` CT_FRONT_SIDED + `(KEEP,KEEP,decr)` CT_BACK_SIDED. So internal volumes cost 4 draws (2 in two-sided mode), external 2 (1) — matches the perf note at docs/dev/plans/2026-06-11-renderer-perf-pass-4.md:111.
- Draw call: `RB_DrawShadowElementsWithCounters(surf, numIndexes)` (tr_render.cpp:186-213) — plain glDrawElements honoring the reduced index count; routes MD5R prim-batch tris to `RB_ARB2_MD5R_DrawShadowElements` (draw_arb2.cpp:1498-1549) which binds the packed family VP (`md5rshadow.vp`, ARB2_MD5R_SHADOW_VOLUME_VPROG_BASE, draw_arb2.cpp:11731), uploads localLight(w=0) + MVP + palette rows, and rebinds VPROG_STENCIL_SHADOW afterwards (1544, 1489). Q4SDK_MD5R is compiled in for non-SDK builds (src/idlib/precompiled.h:87).
- r_showShadows (default 0, RenderSystem_init.cpp:601) debug path draws two-sided with stencil disabled (7276-7311).

## 2. Interaction stencil contract & per-light order

Stencil-path block per light: **src/renderer/draw_arb2.cpp:11599-11649** (taken when the shadow-map classifier rejects the light; with r_useShadowMap=0 default `R_ShadowMapLightWillUseShadowMaps` is always false — tr_light.cpp:1420).
- **Stencil clear per light: scissored, not full** — if `globalShadows || localShadows`: set `backEnd.currentScissor = vLight->scissorRect`, glScissor under r_useScissor, `glClear(GL_STENCIL_BUFFER_BIT)` (11600-11608). Clear value is still the view-level 128 (glClearStencil latched at RB_BeginDrawingView). If no shadows: `glStencilFunc(GL_ALWAYS, 128, 255)` (11609-11614) — interactions draw untested-but-enabled ("invariance rules" comment).
- Order (11616-11630): bind VPROG_STENCIL_SHADOW ("shadow.vp", registered draw_arb2.cpp:11695) when `r_useShadowVertexProgram && R_BindARBProgram(...)` →
  `RB_StencilShadowPass(vLight->globalShadows)` → `RB_DrawMaterialInteractions(vLight->localInteractions)` → re-enable+rebind VP (11620-11621, because interactions disabled/rebound programs) → `RB_StencilShadowPass(vLight->localShadows)` → `RB_DrawMaterialInteractions(vLight->globalInteractions)` → disable VP. Else branch (11625-11630) identical without VP.
- WHY this order (stencil ownership): localInteractions are noSelfShadow receivers — they must be lit while only *global* casters' volumes are in the stencil; localShadows (volumes cast by noSelfShadow surfaces, "don't shadow local surfaces") are stamped afterwards so they only darken globalInteractions, which "get shadows from everything" (tr_local.h:225-228). The GEQUAL/128 exit state of each shadow pass is what the following interaction batch draws under: GL_GEQUAL ref=128 passes where stencil<=128 (unshadowed; volumes net +1 per occluding volume on shadowed pixels).
- Interaction batch state: `GL_State(GLS_SRCBLEND_ONE|GLS_DSTBLEND_ONE|GLS_DEPTHMASK|backEnd.depthFunc)` (draw_arb2.cpp:11227); depthFunc EQUAL for opaque, LESS(→LEQUAL) for translucent.
- Translucent interactions (11636-11646): `glStencilFunc(shadowTranslucentInteractions ? GL_GEQUAL : GL_ALWAYS, 128, 255)` where the bool = r_stencilTranslucentShadows && r_shadows && (globalShadows||localShadows); r_stencilTranslucentShadows default 1, ARCHIVE (RenderSystem_init.cpp:343).
- Loop exit: `glStencilFunc(GL_ALWAYS, 128, 255)` (11653).
- Shadow-map fallback re-enters the identical contract mid-frame: RB_ShadowMapStencilFallbackFiltered (draw_arb2.cpp:10393-10426) — scissored stencil clear + VP bind + two RB_StencilShadowPass calls + interactions.
- Fog/blend lights are skipped by the interaction loop (11494-11499) and drawn by RB_STD_FogAllLights (draw_common.cpp:7762-7816) with **stencil test disabled** (7773), re-enabled at return (7815); the old per-light fog stencil dedup is #if 0 (7782-7805).

## 3. What Vulkan needs

Current VK executor state (src/renderer/Vulkan/vk_GuiExecutor.cpp): dynamic states = viewport, scissor, depth test/write/compareOp, cull mode, front face, depth-bias enable, depth bias (332-342); depth-stencil create-info zeroed = stencil off, comment "stencil stays off until Phase G" (349-352); `rendering.stencilAttachmentFormat = vkCtx.depthFormat` already set (360); depth format probe prefers D24S8 then D32S8 (VulkanDevice.cpp:117-132, comment 113-114 "Stencil is carried now so the Phase G stencil-shadow work needs no format churn"); frame attach clears depth 1.0 / stencil 128 (979-980) and each 3D view re-clears both via vkCmdClearAttachments (1601-1603). Note: only vk_Interactions.cpp exists in src/renderer/Vulkan/ in this working tree — no vk_ShadowMap.cpp yet (F2 presumably in the parallel session).

- **Dynamic stencil state — all five can be dynamic on the existing device with zero new features**: VK_DYNAMIC_STATE_STENCIL_COMPARE_MASK / _WRITE_MASK / _REFERENCE are core Vulkan 1.0 dynamic states; VK_DYNAMIC_STATE_STENCIL_TEST_ENABLE and VK_DYNAMIC_STATE_STENCIL_OP come from VK_EXT_extended_dynamic_state which is promoted to core 1.3 with functionality mandatory (no feature bit) — same promotion family as the CULL_MODE/FRONT_FACE/DEPTH_* dynamics the executor already uses (and DEPTH_BIAS_ENABLE from EDS2, also mandatory-in-1.3). Device creation enables only dynamicRendering, synchronization2, samplerAnisotropy (VulkanDevice.cpp:521-535). [Spec claims from knowledge; not re-verified against the spec text.]
- **Dedicated shadow pipeline required**: GLS_COLORMASK|GLS_ALPHAMASK (color writes off) maps to colorWriteMask=0, which is NOT dynamic without EDS3 — so one new pipeline with colorWriteMask 0, no blend, the vec4 vertex input, and the stencil dynamics added to pDynamicStates. Depth: test on, write off, compare LESS_OR_EQUAL — all already dynamic.
- **Vertex input**: binding stride 16, single attribute VK_FORMAT_R32G32B32A32_SFLOAT @0 (shadowCache_t). Vertex shader = shadow.vp equivalent: `clip = MVPfixed * (v.w==1 ? v : vec4(v.xyz - lightOrigin, 0))`; per-space MVP (VK_FixupClipSpaceZ applied) + per-space localLightOrigin uniform, mirroring env[4] uploads (draw_common.cpp:7214-7221).
- **Two-sided**: separate front/back VkStencilOpState is core 1.0; INCREMENT_AND_WRAP/DECREMENT_AND_WRAP are core — the GL gate (`glStencilOpSeparate && INCR_WRAP`, draw_common.cpp:7420-7423) is unconditionally satisfiable, so the single-pass CT_TWO_SIDED path can be the only implementation (never the 2x cull-flipped fallback). Face mapping under the executor's established conventions (negative-height viewport + FRONT_FACE_COUNTER_CLOCKWISE; CT_FRONT_SIDED ≡ VK_CULL_MODE_FRONT_BIT non-mirror, vk_GuiExecutor.cpp:1627-1628): GL_BACK == VK back face, so `frontSidedFace` ops (draw_common.cpp:7321) go to VK_STENCIL_FACE_BACK_BIT in non-mirror views, flipped for mirrors.
- **Infinite projection / caps**: R_SetupProjection is *unconditionally* infinite-far — matrix[10]=-0.999, [14]=-2*zNear, [11]=-1, [15]=0 with an explicit "crunch so w=0 vertexes do not rasterize right at the wraparound point" comment (tr_main.cpp:940-951); r_useInfiniteFarZ is declared (RenderSystem_init.cpp:376) but never read anywhere. Through the VK fixup row2'=(row2+row3)/2 (vk_GuiExecutor.cpp:183-190): a w=0 extruded vert has clip.w=-z_e, old clip.z=-0.999·z_e → new z=-0.9995·z_e → z/w=0.9995 < 1, so far caps at infinity stay strictly inside the [0,1] clip volume — z-fail rear caps survive the fixup unchanged. [Derived math, not yet empirically tested.]
- **depthClamp: not needed for parity** — the GL path never uses depth clamp; near-plane robustness comes from the z-fail preload and far-plane robustness from the infinite projection. VkPhysicalDeviceFeatures.depthClamp is optional and currently not enabled; leave it off.
- **Depth bounds test: skip in Phase G** — pure fill-rate optimization (draw_common.cpp:7271-7273, gated on glConfig.depthBoundsTestAvailable). On VK it would need the optional depthBounds feature (not enabled) + VK_DYNAMIC_STATE_DEPTH_BOUNDS(+_TEST_ENABLE). The zmin/zmax values are GL window-z and remain valid because the fixup preserves window depth.
- **Depth bias**: enabled during the shadow pass by default (units +1, factor 0) — already dynamic in the executor.
- **Per-light scissored stencil clear**: vkCmdClearAttachments(VK_IMAGE_ASPECT_STENCIL_BIT, value 128, rect = vLight->scissorRect offset by viewport) inside the render pass mirrors draw_arb2.cpp:11600-11608.
- Interaction pass hook: VK_Interactions_DrawLights (src/renderer/Vulkan/vk_Interactions.cpp:613-707) currently draws local then global interactions unshadowed per light (699-700) — Phase G interleaves the two shadow passes + stencil funcs into that loop, plus the translucent stencil-func selection (11636-11643 parity).

## 4. r_useShadowVertexProgram / VPROG_STENCIL_SHADOW semantics

- Defaults: r_useShadowVertexProgram 1 (RenderSystem_init.cpp:287), r_useTurboShadow 1 (341) — **GPU extrusion is the codebase default**. The VP ("shadow.vp", VPROG_STENCIL_SHADOW, prog table draw_arb2.cpp:11695) is a retail Q4 pak asset — NOT present in repo content (content/baseoq4/pak0/glprogs has no shadow.vp). Its contract is pinned by the data: verts with w=1 pass through unchanged (prelight/CPU-projected verts), verts with w=0 are projected to infinity along (vertex.xyz − program.env[4].xyz) — env[4] is uploaded with w=0 (draw_common.cpp:7218) so the result keeps w=0 (homogeneous direction). [Exact ARB text unavailable in-repo; semantics are certain, wording speculative.]
- **Critical asymmetry**: the volume *topology* is baked at interaction time by the same cvar (tr_turboshadow.cpp:683) — VP-flavored caches contain unprojected w=0 verts, so a backend that can't extrude in the vertex shader cannot consume them. The VK backend must implement VP-equivalent extrusion (trivial in SPIR-V); the CPU/fixed-function branch (draw_arb2.cpp:11625-11630) only works because CPU-created caches carry pre-projected verts.
- MD5R packed prim-batch shadows are a separate family: md5rshadow.vp + palette rows + own vertex/index streams (draw_arb2.cpp:1384-1549); compiled in via Q4SDK_MD5R (idlib/precompiled.h:87). Only relevant where primBatchMesh assets are live; can be a follow-up slice (RB_T_Shadow already bypasses the classic stream for them, draw_common.cpp:7212, 7226).

## 5. Fastest correct retail-default subset

Defaults recap: r_shadows 1, r_useShadowMap 0, r_useShadowVertexProgram 1, r_useTwoSidedStencil 1 (342), r_useExternalShadows 1 (561), r_useTurboShadow 1, r_useOptimizedShadows 1 (562), r_useDepthBoundsTest 1 (565), r_stencilTranslucentShadows 1 (343), r_shadowPolygonFactor 0 / r_shadowPolygonOffset -1 (523-524).

Minimum for correct retail-default shadows on VK:
1. One stencil-volume pipeline: vec4@stride16 input, colorWriteMask 0, extrusion vertex shader, stencil dynamics (op/testEnable/compareMask/writeMask/reference) added; depth LEQUAL, write off, bias on (0, +1).
2. Single-pass two-sided ops only — always available on VK (wrap ops + separate faces core), including the `!external` preload draw pair; skip the two-pass fallback entirely.
3. The per-surface index-count/external selection verbatim from draw_common.cpp:7238-7268 — all inputs (numShadowIndexesNoCaps/NoFrontCaps, DSF_VIEW_INSIDE_SHADOW, viewInsideLight, viewSeesShadowPlaneBits, shadowCapPlaneBits, SHADOW_CAP_INFINITE) are front-end data already computed for the VK backend.
4. Per-light: scissored stencil clear to 128 → globalShadows → localInteractions(GEQUAL/128) → localShadows → globalInteractions(GEQUAL/128) → translucent (GEQUAL if shadowed else ALWAYS, depth LEQUAL); ALWAYS/128 for shadow-less lights; per-surface scissor + per-space matrix/light-origin inside the shadow chain walk.
5. Skippable initially: depth-bounds (perf only, needs optional feature), r_showShadows debug modes, MD5R packed streams, fog/blend lights (stencil-off, separate Phase G slice), shadow-map/stencil fallback interplay (dormant at r_useShadowMap 0).
