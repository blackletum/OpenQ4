### Render targets, formats, layout (GL reference)
- **Projected atlas**: TT_2D `FMT_DEPTH` (GL_DEPTH_COMPONENT, `OpenGL/gl_Image.cpp:420-424`), size = `r_shadowMapAtlasSize` (default 4096, clamp 2048-8192 ∧ maxTextureSize), 1 mip, depth-only FBO, filter TF_NEAREST/TR_CLAMP (draw_arb2.cpp:5716-5736). Cell = `r_shadowMapSize` (1024 default); CSM lights take a 2×2 cell block. TEXTURE_COMPARE_MODE toggled at bind: hardware LEQUAL compare + LINEAR (2×2 PCF) when `r_shadowMapDepthCompare` (default 1), NEAREST raw depth for manual/PCSS path (:5775-5786).
- **Projected scratch**: TT_2D FMT_DEPTH, size = tileSize×atlasDiv per light; `g_shadowMapRenderTexture` flips between scratch and atlas depending on cache state (:5744-5768).
- **Point light**: cube color `FMT_RGBA8` (packed radial depth; `FMT_RGBA16F` if `r_shadowMapPointHighPrecision`) + cube `FMT_DEPTH`, face size `r_shadowMapPointSize` (512). Default receiver samples the **depth cube with hardware compare** (`r_shadowMapPointDepthCompare` 1); the color cube is the manual fallback. Seamless cubemap enabled (:5803-5876).
- **Translucent moments** (default OFF, `r_shadowMapTranslucentMoments 0`): 3× RGBA16F MRT (:5891-5927) — skippable for Phase F.

### Caster pass mechanics (`RB_RenderShadowMap` :7795-7984)
- One GLSL caster program for all opaque+perforated casters; **slope-scale/constant depth offset applied in the shader** (`r_shadowMapPolygonFactor` 0.75 / `r_shadowMapPolygonOffset` 0.5) because shader-written depth ignores glPolygonOffset (:7866-7883). (On Vulkan, dynamic depth-bias state is an alternative if the receiver convention stays consistent — speculation/design choice.)
- State: colorMask off, depth LEQUAL+write, blend/stencil off, scissor on (:7885-7891).
- Per cascade: viewport+scissor to the tile (slot origin + `cascadeIndex%atlasDiv × tileSize`), `glClear(DEPTH)` per tile, projection = `R_ShadowMapClipPlanesToGLMatrix(clipPlanes[cascade])` (:7895-7923). Caster chains: primary/secondary static, tertiary/quaternary dynamic; compose mode blits cached static tiles then draws dynamics only (:7852-7864).
- Caster culling: `r_shadowMapCasterCulling` default 2 = store back faces (:305, apply :2907).
- Point pass (:7986-8160): 6 faces, per-face FBO bind, colorMask ON, clear color 1.0, near=clamp(0.5, depthClamp?16:4, far×0.01), depth clamp when available, per-face view matrices (:7303-7349).

### Receiver (shadow interaction) uniform surface (:9822-9971, :8549-8615)
Per-light: samplers 0-4 material, 5 shadow map, 6-8 moments; shadowTexelSize, shadowBias, shadowNormalBias, texelDepthBias[4], normalOffsetWorld[4], receiverPlaneBias, filterRadius/Taps/Mode, PCSS radii, **composed atlasRect[4]** (slot composition at upload :9899-9918), splitDepths[4], cascadeBiasScale[4], cascadeCount, cascadeBlend. Per-model-space: shadow matrix as 4 rows × 4 cascades of vec4 — clip planes transformed world→model-local CPU-side (:8552-8581). Per-draw: localLightOrigin, localViewOrigin, lightProjection S/T/Q + falloff S, bump/diffuse/specular 2×vec4 matrices, diffuseColor, specularColor, vertexColor modulate/add (:8584-8615).

## 2. Cvar surface + off behavior
- `r_useShadowMap` **default "0"** (`RenderSystem_init.cpp:288`, CVAR_ARCHIVE|BOOL). The flip to 1 is still pending user sign-off (per project memory). Full family: :288-332 and :525-546. Key: `r_shadows` 1 (:551, the kill switch — there is no `r_skipShadows`), `r_skipInteractions` (:465), `r_shadowMapCSM` 0 (:289), `r_shadowMapPointLights` 1 (:315 — "point lights are the dominant light class in Quake 4"), `r_shadowMapSkipStencilShadows` 1 (:321), `r_shadowMapAtlasSize` 4096 (:317), `r_shadowMapSize` 1024 (:525), `r_shadowMapPointSize` 512 (:316).
- **Shadow maps off (r_useShadowMap 0)**: Interaction.cpp:2016 gate means caster chains are NOT built (zero cost); stencil volumes built normally; backend uses the retail stencil branch of `RB_ARB2_DrawInteractions` (draw_arb2.cpp:11599-11630).
- **Shadow maps on**: caster chains built; stencil volumes skipped per light only when `R_ShadowMapLightWillUseShadowMaps` passes (tr_light.cpp:1419-1448) — which requires `RB_ShadowMapResourcesKnownGood` (backend query). Under vk this stub returns false (vk_Backend.cpp:765), so **stencil lists are always still built under vk regardless of cvars** — correct/safe for Phase F (and needed for Phase G), just wasted front-end CPU until vk implements the real check.

## 3. Minimal Phase F shape — verified against the GL reference
**Step F1: interactions with all lights unshadowed.** The GL path supports this cleanly in both branches:
- Retail branch: lights with no shadow surfs skip the stencil clear and set `glStencilFunc(GL_ALWAYS,128,255)` then draw interactions (draw_arb2.cpp:11609-11614).
- Shadow-map branch: `RB_ShadowMapRunPass` with no casters/shadow surfs → stencil ALWAYS + `RB_DrawMaterialInteractions` (:10872-10889); render/mask failure → stencil fallback (:11077-11078).
- Since Phase E's vk pipelines have stencil disabled entirely, drawing every light's local/global/translucent interactions with stencil off is exactly the GL "unshadowed light" behavior applied to all lights. Loop skeleton = RB_ARB2_DrawInteractions: skip `IsFogLight()`/`IsBlendLight()` (:11494-11499), skip empty lights (:11501), opaque interactions at depthFunc EQUAL additive ONE:ONE (:9827 shows `GLS_SRCBLEND_ONE|GLS_DSTBLEND_ONE|GLS_DEPTHMASK|backEnd.depthFunc`), translucent at depthFunc LESS (:11563-11589).
- Must port from excluded TUs: the decomposition walk (tr_render.cpp:875 `RB_CreateSingleDrawInteractionsFiltered` + :782 `R_SetDrawInteraction` + :836 `RB_SubmittInteraction` — includes r_skipBump/Diffuse/Specular substitutions and no-op culling) and one SPIR-V interaction shader ≈ interaction.vfp semantics (param list at draw_arb2.cpp:11098-11169).
- Walk vLight chains directly (GL-parity path); do NOT depend on ModernShadowPlanner (not wired under vk).

**Step F2: shadow maps, scratch-first.** Reuse included TUs for all math (`R_ClassifyShadowMapLight`, `R_BuildShadowMapProjectedLightState`, `R_ShadowMapProjectedFilterSettings`). Defer: static caches/atlas-slot reuse/compose (`SHADOWMAP_RENDER_COMPOSE_DYNAMIC`), translucent moments (default off), CSM (default off), budget admission, subview policy — render every shadow-relevant light fresh each view (GL does this when caches miss; only a perf delta). Failure/no-caster outcomes: GL falls back to stencil; vk has no stencil until Phase G, so failures render unshadowed — visible divergence only for lights that would have shadowed, acceptable interim while `r_useShadowMap` defaults 0 under vk.
- Point lights are required for the milestone to look right (dominant light class, :315); projected+point both needed; parallel lights route through projected (tr_light.cpp:1439-1441).

## 4. vk_GuiExecutor frame-shape changes
Current: ONE dynamic-rendering scope per frame — `vkCmdBeginRendering` at BeginFrame (vk_GuiExecutor.cpp:821; swapchain color loadOp CLEAR :793, per-slot depth loadOp CLEAR :807), closed at EndFrameAndPresent (:851). All 2D/3D views record inside it; per-3D-view depth reset via `vkCmdClearAttachments` (:1362-1376). Pipelines target swapchain+depth formats via `VkPipelineRenderingCreateInfo` (:331-337). Frame slots: `VK_FRAMES_IN_FLIGHT=2` (VulkanDevice.h:33), depth format D24S8/D32S8 probe (VulkanDevice.cpp:118-124).

Required changes:
1. **Split the scope**: factor `VK_Exec_BeginMainRendering(bool clear)` / `VK_Exec_EndMainRendering()`. Resume after a shadow interruption must use **loadOp LOAD for color and depth** (a new path — today the only open always clears) and re-establish baseline dynamic state (:825-831).
2. **Per-3D-view sequence**: end main rendering → barrier atlas (UNDEFINED→DEPTH_ATTACHMENT_OPTIMAL) → one dynamic-rendering scope on the atlas (depth-only, no color attachment; per-light/per-cascade viewport+scissor tiles; per-tile clears via vkCmdClearAttachments) → end → barrier atlas → DEPTH_READ_ONLY/SHADER_READ (fragment) → per point light: 6 short scopes on cube-face views (color RGBA8 + depth face) or multiview → barriers → resume main rendering (LOAD). Rendering ALL lights' maps in one interruption before the interaction loop = one suspend/resume per view; this pushes vk toward **atlas tiles from day one** (per-light slots in one image) rather than GL's single reused scratch texture, which would force per-light interruptions.
3. **Atlas resources are module-owned VkImages** (not idImage): one D24S8-or-D32 atlas (can reuse `vkCtx.depthFormat` for pipeline-format simplicity), single copy (not per-frame-slot — produced and consumed inside the same command buffer, barriers serialize; no CPU hazard). Point: per-light-in-flight cube depth+color images or a small pool.
4. **New pipelines** (new `VkPipelineRenderingCreateInfo` variants): caster depth-only (0 color attachments, atlas depth format), point caster (RGBA8 + depth), interaction (swapchain formats, additive ONE:ONE blend, depth EQUAL, variants: unshadowed/projected-shadow/point-shadow). The existing single 128B-push layout (:594-597) cannot serve interactions — new pipeline layout needed.
5. **Descriptors**: current layout = one combined-image-sampler (:562-577) with per-image cache (:479-521). Interactions need 5 material textures + shadow map + comparison sampler (no compare samplers exist in vk_Image.cpp today) + a UBO. Suggested: set 0 = existing per-image (keep for 2D), new set layout for interactions (5 samplers + shadow sampler + 2 dynamic UBOs), or bind the 5 material images as 5 sets of the existing layout (matches GL's 5 units but 5 binds/draw).
6. Vertex ring (32MB :66) + triMemo already dedupe caster geometry re-streaming (casters are the same ambient tris as depth/ambient walks, :923-955).

## 5. Uniform budget: push constants overflow — UBO ring recommended
- Current push block = exactly **128B** (`vkGuiPushConstants_t` :91-97 = mvp 64 + 4×vec4 64; range :594-597). 128B is the Vulkan guaranteed `maxPushConstantsSize` minimum.
- Unshadowed interaction per-draw payload (drawInteraction_t tr_local.h:367-390; uploads draw_arb2.cpp:11098-11169): MVP 64 + localLightOrigin 16 + localViewOrigin 16 + lightProjection[4] 64 + bumpMatrix 32 + diffuseMatrix 32 + specularMatrix 32 + diffuseColor 16 + specularColor 16 + vertexColorParams ~8 ≈ **296B ≈ 2.4× the 128B guarantee** (over 256B too, so even the common desktop extended size doesn't cover it — vendor-size claim is speculation).
- Shadow receiver adds per-light ≈ 600-700B std140 (shadowRows 4×4×vec4=256B :8556-8581, atlasRect 64B, splitDepths/biasScale/texelDepthBias/normalOffsetWorld as vec4-arrays ≈256B, scalars ~64B :9857-9948), and shadowRows change **per model space**, not per light.
- **Recommendation**: keep the 128B push range for per-draw volatile data (MVP 64B + a few params/UBO offsets); stream everything else through a new host-visible uniform ring (same `vkRing_t` pattern :72-78, usage UNIFORM_BUFFER_BIT, 256B alignment — `minUniformBufferOffsetAlignment` ≤ 256 guaranteed) bound as VK_DESCRIPTOR_TYPE_UNIFORM_BUFFER_DYNAMIC with dynamic offsets: binding A = per-draw interaction block (origins/projections/stage matrices/colors, ~256B slice), binding B = per-light shadow block (~768B slice, rewritten per space for shadowRows). Optional simplification (speculation): upload world-space shadow rows once per light plus the 64B model matrix per draw and do the world→local plane transform in-shader, eliminating the per-space CPU recompute at :8567-8576.
