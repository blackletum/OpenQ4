## 2. How interactions arrive

- `idInteraction::CreateInteraction` (src\renderer\Interaction.cpp:1488-1663): per model surface fills `surfaceInteraction_t` (Interaction.h:49-74: `lightTris`, `shadowTris`, `ambientTris`, `shader`, shadow-LOD memo, `cullInfo`). Light tris created via `R_CreateLightTris` (Interaction.cpp:847+) or deferred as `LIGHT_TRIS_DEFERRED` (:1567-1575). Shadow-map caster eligibility per surface at :1584-1620; dynamic-model stencil volumes suppressed while the light will render shadow maps (`suppressDynamicShadowVolume` :1626-1633); stencil volumes via `R_CreateShadowVolume` :1639.
- `R_CreateLightTris` (Interaction.cpp:847-1000): light tris **reference the ambient surface's verts** (`newTri->numVerts = tri->numVerts; R_ReferenceStaticTriSurfVerts` :883-884, `ambientSurface` back-pointer :880); indexes are a backface/frustum-culled subset of the ambient indexes.
- `idInteraction::AddActiveInteraction` (Interaction.cpp:1749-2135), called from tr_light.cpp:2859:
  - Ensures `ambientTris->ambientCache` via `R_CreateAmbientCache` (:1889) then shares it into lightTris (`lightTris->ambientCache = ambientTris->ambientCache` :1895) — so **interaction receivers draw from the ambient surface's idDrawVert cache with the lightTris index subset**. Index cache alloc at :1926-1940.
  - Receiver routing (:1966-1975): MC_TRANSLUCENT → `translucentInteractions`; else noSelfShadow material (and light casts shadows) → `localInteractions`; else `globalInteractions` — via `R_LinkLightSurf` (tr_light.cpp:1098-1150), which evaluates surface registers (:1135) and sets scissor = light∩entity rect (:1842-1843).
  - Shadow-map casters (:2016-2101): gate = `r_useShadowMap && (!pointLight || parallel || r_shadowMapPointLights)`; per-surface policy (spectrum match :2031, emitter-panel reject :2028, LOD admission `R_CachedInteractionShadowLODAdmitted` :2046) → `R_EnsureShadowMapCasterCaches` (Interaction.cpp:403-427: guarantees `ambientCache` (full idDrawVert) + static `indexCache`) → linked with `R_LinkShadowMapCasterSurf` (Interaction.cpp:371-397). Casters split static/dynamic by `R_ShadowMapCasterIsDynamic` (:2072).
  - Stencil shadow surfs (:2103-2133) skipped when `R_ShadowMapLightWillUseShadowMaps` (Phase G otherwise).
- **Caches per interaction surface**:
  - `ambientCache` — full `idDrawVert[numVerts]` (Model.h:149; created tr_light.cpp:242-259, tangents derived on demand via `R_DeriveTangents` :250-252). This is the ONLY vertex cache interaction passes read.
  - `lightingCache` — `lightingCache_t { idVec3 localLightVector }` (Model.h:64-67), one per ambient vert; **non-vertex-program cards only**: `R_CreateLightingCache` (tr_light.cpp:385-445) early-returns true when `tr.backEndRendererHasVertexPrograms` (:398). Under vk, `glConfig.allowARB2Path=true` (vk_Backend.cpp:98) makes SetBackEndRenderer pick BE_ARB2 with `backEndRendererHasVertexPrograms=true` (RenderSystem.cpp:1223-1226), so **lightingCache is never allocated — the vk backend must never reference it**. Same gate skips `R_SpecularTexGen` (tr_light.cpp:1138-1144).
  - `shadowCache` — `shadowCache_t { idVec4 xyz }` (Model.h:69-71), 2 verts per source vert, `R_CreateVertexProgramShadowCache` (tr_light.cpp:470-510) — Phase G stencil path only.

## 3. Light shader stage walk (the reference the vk interaction pass must replicate)

`RB_CreateSingleDrawInteractionsFiltered` (src\renderer\tr_render.cpp:875-1033) — **tr_render.cpp is EXCLUDED from the vk module** (tools\build\meson_sources.py:112-126 excludes draw_arb2.cpp, draw_common.cpp, tr_backend.cpp, tr_render.cpp, tr_rendertools.cpp, ModernGLExecutor.cpp, ModernGLShaderLibrary.cpp, RenderGraphResources.cpp, GLStateCache.cpp, GLDebugScope.cpp, RendererUpload.cpp), so vk needs its own GL-free re-implementation. Semantics:
- Per surf: localLightOrigin/localViewOrigin via `R_GlobalPointToLocal` (w=0/w=1, :918-921); base `lightProject[4]` planes → surface-local (:925-928); `inter.ambientLight = lightShader->IsAmbientLight()` (:922); `lightFalloffImage = vLight->falloffImage` (:916).
- Outer loop over light stages (:930-1027): skip if `!lightRegs[stage->conditionRegister]` (:936); `inter.lightImage = lightStage->texture.image` (:940); if `lightStage->texture.hasMatrix`, bake the light texture matrix into the S/T/Q texgen planes (`RB_BakeTextureMatrixIntoTexgen`, :944-947); lightColor = `backEnd.lightScale * lightRegs[stage->color.registers[0..2]]`, alpha unscaled (:959-962).
- Inner loop over surface stages by `surfaceStage->lighting`: `SL_AMBIENT` ignored (:972); `SL_BUMP` flushes pending interaction then sets bumpImage+bumpMatrix (:976-986); `SL_DIFFUSE`/`SL_SPECULAR` set image+matrix+color (× lightColor) and `vertexColor` (:988-1021); multiple bump/diffuse/specular stages ⇒ multiple primitive interactions per light stage. Stage data extracted by `R_SetDrawInteraction` (tr_render.cpp:782-829: 2×idVec4 texture matrix rows, color clamp 0..1).
- `RB_SubmittInteraction` (:836-861): defaults diffuse/specular→blackImage, bump→flatNormalMap; skips draw if bump missing or both colors black.
- `drawInteraction_t` (tr_local.h:367-390): surf, 5 images, diffuse/specularColor, `stageVertexColor_t vertexColor`, `ambientLight`, localLightOrigin/ViewOrigin, `lightProjection[4]`, bump/diffuse/specularMatrix[2].
- GL consumption reference `RB_ARB2_DrawInteraction` (draw_arb2.cpp:11086-11208): vertex env params PP_LIGHT_ORIGIN/PP_VIEW_ORIGIN/PP_LIGHT_PROJECT_S,T,Q/PP_LIGHT_FALLOFF_S/6 matrix rows; SVC modulate/add packing (:11121-11159); fragment env 0 = diffuseColor, env 1 = specularColor×2 (:11162-11169); textures 1=bump 2=falloff 3=projection 4=diffuse 5=specular (:11174-11191); polygon offset for MF_POLYGONOFFSET decals (:11194-11207).
- Per-light backend loop `RB_ARB2_DrawInteractions` (draw_arb2.cpp:11458-11666): skip fog/blend lights (:11494-11499), skip if no interaction chains (:11501-11504); shadow-map-supported lights run `RB_ShadowMapRunPass(LOCAL→globalCasters only; GLOBAL→global+local casters)` then translucent receivers (:11549-11589); unsupported lights fall back to stencil order globalShadows→localInteractions→localShadows→globalInteractions (:11616-11630). `backEnd.lightScale` comes from `RB_DetermineLightScale` (tr_render.cpp:675-726, also excluded — vk must port it: max stage color over all viewLights vs `tr.backEndRendererMaxLight`=999).

## 4. Shadow-map planner outputs + glConfig gates

Two separate systems:

**(a) ModernShadowPlanner (src\renderer\ModernShadowPlanner.h/.cpp)** — compiled INTO the vk module (not excluded) but only *invoked* from ModernGLExecutor.cpp:6816 etc. (excluded) and self-tests (ModernClusteredLighting.cpp:2616,2742) — i.e. under vk it currently never runs in the frame loop. Its per-light output `modernShadowLightDescriptor_t` (ModernShadowPlanner.h:81-201) is the richest "what a modern backend needs" contract: mapType (PROJECTED/POINT/CASCADE), policy, fallbackReason, priority/fairness, resolution, tileCount/cascadeCount/atlasDiv, `atlasRect[4]` + `tileAtlasRect[6][4]`, `shadowMatrix[16]`, `projectedBaseClipPlanes[4][4]`, per-cascade clip planes/atlas rects/split depths/bias/texel sizes (:121-137), caster/receiver counts (:111-116), point atlasDiv=3 six-face layout invariant (ModernShadowPlanner.cpp:962). Gates: `R_ModernShadowPlanner_SupportReason` (ModernShadowPlanner.cpp:985-1022) checks r_shadows, noShadows flags, ambient/no-cast shaders, receivers, `r_useShadowMap`, `glConfig.maxTextureUnits/maxTextureImageUnits >= 6` (:1015) and point lights need `glConfig.cubeMapAvailable` (:1018). Translucent moments need `glConfig.GLSLProgramAvailable && maxTextureUnits>=9 && maxTextureImageUnits>=9 && maxDrawBuffers>=3 && maxColorAttachments>=3` (:435-442); modern receiver sampling additionally `maxTextureImageUnits >= 13` + hasUBO (:444-454).

**(b) The ARB2 backend's own atlas/cache machinery** lives in draw_arb2.cpp (excluded from vk): atlas grid `RB_ShadowMapAtlasDivForLight`/`RB_ShadowMapAtlasFindFreeBlock` (:3191, :3564-3572), pass driver `RB_ShadowMapRunPass`, support `RB_ShadowMapLightSupportReason` (:6002, gates at :6012 texture units, :6024 cubeMap). **GL-free shared helpers the vk backend CAN reuse directly** (both compiled into vk, no glConfig/gl calls):
- `ShadowMapClassification.cpp:53-93` `R_ClassifyShadowMapLight`: point (pointLight && !parallel) → atlasDiv 3, tileCount 6; CSM (r_shadowMapCSM, cascades clamp 1-4) → atlasDiv 2; else 1/1. Parallel lights carry pointLight=true but route through projected machinery (:58-62).
- `ShadowMapProjected.h:40-80` `R_BuildShadowMapProjectedLightState`: full projected/cascade state — baseClipPlanes from `vLight->lightProject` (`R_ShadowMapBuildClipPlanes` :70) or synthesized ortho for parallel (:71), `baseShadowMatrix[16]` (`R_ShadowMapClipPlanesToGLMatrix` :73), per-cascade splitDepths/biasScale/texelDepthBias/atlasRect (`R_ShadowMapBuildAtlasRect` :74), stability quantization :76-77.

**Front-end behavior keyed on glConfig that flips when vk pins caps** — vk currently pins (vk_Backend.cpp:62-110): maxTextureUnits=8, maxTextureCoords=8, maxTextureImageUnits=16, cubeMapAvailable=true, allowARB2Path=true, NPOT/S3TC/BPTC=true. NOT pinned (stay 0/false): GLSLProgramAvailable, maxDrawBuffers, maxColorAttachments, ARBVertexProgram/FragmentProgramAvailable, depthBoundsTestAvailable.
1. `R_ShadowMapLightWillUseShadowMaps` (tr_light.cpp:1419-1448) — the front-end mirror that **elides stencil volume generation+linking** when a light will shadow-map. Its glConfig gates (maxTextureUnits/ImageUnits>=6 :1436, cubeMapAvailable for point :1443) already PASS under the vk pins; the only closed gate is the module hook `RB_ShadowMapResourcesKnownGood(pointLightPath)` (:1447, declared tr_local.h:1109) whose **vk stub returns false** (src\renderer\Vulkan\vk_Backend.cpp:765-768). Consequence today: with r_useShadowMap on under vk, the front-end builds shadow-map caster chains AND still builds/links stencil volumes. The moment Phase F makes the stub return true (with defaults r_shadowMapSkipStencilShadows=1 RenderSystem_init.cpp:321, r_shadowMapMaxUpdatesPerView=0 :320), the front-end starts suppressing dynamic shadow volumes (Interaction.cpp:1626-1633), skipping stencil links (Interaction.cpp:2108), and skipping prelight volumes (tr_light.cpp:1464). Hazards: (i) the stub must be per-light-class honest — it takes `pointLight` and must report point support separately; (ii) the sticky per-light fallback `lightDef->shadowMapStencilFallbackSticky` (tr_light.cpp:1427) is the restore contract when a backend pass fails mid-frame — vk must set it exactly like the GL backend or lights lose shadows permanently until the next regeneration; (iii) r_shadowMapMaxUpdatesPerView>0 intentionally disables elision (:1424-1426) — a vk update-budget feature must keep that coupling.
2. Translucent moment casters: `R_TranslucentShadowMapMomentsSupportedForLight` (Interaction.cpp:109-117) needs GLSLProgramAvailable+9 units+3 draw buffers/attachments — all false/0 under vk, so translucent chains stay empty and translucent surfaces fall to the stencil-parity opaque-caster path (Interaction.cpp:1591, 2045). If vk ever pins GLSLProgramAvailable/maxDrawBuffers/maxColorAttachments for other reasons, the front end silently starts building translucent moment chains + `SHADOWMAP_CASTER_REJECT_TRANSLUCENT_*` accounting — pin deliberately.
3. `R_CreateLightingCache`/`R_SpecularTexGen` skipped because backEndRendererHasVertexPrograms=true (see §2) — depends on allowARB2Path pin; do not unpin.
4. `r_shadowMapConservativeCasters` (default 1, RenderSystem_init.cpp:300) + `R_ShadowMapConservativeCastersEnabled` (Interaction.cpp:429-431): keeps caster-only interactions alive for off-screen casters (AddActiveInteraction :1761-1816) — cvar-gated only, already active under vk once r_useShadowMap is on. r_useShadowMap default is still "0" (RenderSystem_init.cpp:288; flip pending user sign-off per project memory).

## 5. Vertex formats per pass

`idDrawVert` (src\idlib\geometry\DrawVert.h:30-84): 64B; xyz@0 (3f), color@12 (4×u8), normal@16 (3f), color2@28 (4×u8), tangents[0]@32 (3f), tangents[1]@44 (3f), st@56 (2f). Offsets pinned by asserts :76-84.
- **Interaction pass** (classic ARB2 binding, draw_arb2.cpp:11326-11331; GLSL material variant :9753-9758): position(xyz@0), color(4×u8@12, for SVC modulate), attr8=st@56, attr9=tangent0@32, attr10=tangent1@44, attr11=normal@16. **color2 is NOT read** by any interaction path (MD5R packed path uses its own rvSilTraceVert buffers). Source buffer = shared `ambientCache`, indexes = lightTris subset.
- **Shadow-map caster pass** (opaque: draw_arb2.cpp:7133-7134): position only from `ambientCache`; perforated casters additionally sample st for alpha-test (`RB_ShadowMapDrawPerforatedCasterProgram` :7139); translucent moment casters read xyz+st+color (:7179-7182). Casters draw ambient geometry (full tri->indexes), never lightTris.
- **Stencil shadows** (Phase G): `shadowCache_t` idVec4 xyz only (draw_common.cpp:7492-7493).
- Custom-GLSL lit surfaces additionally bind classic normal/texcoord/color client arrays from the same idDrawVert (draw_arb2.cpp:9677-9684).
