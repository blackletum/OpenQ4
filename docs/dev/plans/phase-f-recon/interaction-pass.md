## 3. Interaction shader math (GLSL equivalents to reproduce in SPIR-V)

`material_interaction.vs` (glprogs, 64 lines): tangent-space transform via dot with attr_Tangent/Bitangent/Normal rows; `vLightVector = TS(toLight)`, `vHalfAngleVector = TS(normalize(toLight)+normalize(toView))`, `vViewVector = TS(toView)`; texcoords = dot(texCoord, S/T rows); `vLightFalloffTexCoord = (dot(pos,falloffS), 0.5, 0, 1)`; `vLightProjectionTexCoord = (dot S, dot T, 0, dot Q)`; `vVertexColor = gl_Color.rgb*modulate + add`; position via ftransform (MVP).

`material_interaction.fs` (95 lines) — **stock path (uStockInteraction>0.5) is the interaction.vfp parity math**: normal = `bumpSample.agb*2-1` (DXT5/RXGB swizzle: alpha=x, green=y, blue=z); `ndotl = max(dot(normalize(vLightVector), N), 0)`; `light = ndotl * texture2DProj(falloff) * texture2DProj(lightProjection)`; diffuse = tex*uDiffuseColor; **stock specular = clamp(dot(H_interpolated_unnormalized, N)*4 - 3, 0, 1)² × 2** (no specular table lookup in GLSL parity; ARB2 binds specularTableImage on unit 6 for the .vfp); `color = (diffuse + specular*uSpecularColor) * light * vVertexColor`, alpha 0. Ambient lights (uAmbientLight): lightDir from `ambientNormalMap` cube sample at (0,0,1). Enhanced (non-stock) path adds normal-scale Z reconstruction, gloss-driven power 10-40, fresnel, boost (32-57).

`shadow_interaction.vs` (100 lines) adds: `uShadowRow0..3[4]` = per-cascade shadow matrix rows **localized to model space**, `uShadowNormalOffsetWorld[4]`; per cascade `vShadowCoordN = rows · (pos + geometricNormal * offsetWorld * sinTheta)` (normal-offset anti-acne, 55-62,90-93); `vShadowLightCos = max(dot(N, lightDir),0)`; `vViewDepth = -viewPos.z` (cascade select key).

`shadow_interaction.fs` (820 lines) — receiver: cascade = first `vViewDepth < uShadowSplitDepths[i]` (533-545); `SampleShadowCascade` (357-437): reject invalid w (NaN, <1e-5, >65536 → lit); `localUv = (xy/w)*0.5+0.5`, **depth = coord.z UNdivided** (the row construction produces [0,1] depth directly); outside [0,1] uv/depth → lit (no shadow outside cascade); atlas uv = `atlasRect.xy + localUv*(zw-xy)` clamped to guard band (`texelSize * (filterRadius+0.75)`, 221-224); bias (280-297) = max(scalarBias=(uShadowBias + uShadowNormalBias*sinTheta)*cascadeBiasScale, texelBias*(1+slope), receiver-plane bias = dFdx/dFdy depth gradient * filterRadius); compare: `#ifdef OPENQ4_SHADOW_COMPARE` → `shadow2D(sampler2DShadow, (uv, depth-bias))` (HW compare, LEQUAL) else manual `texture2D` step; **PCF**: 1/5/9/13 fixed Poisson taps (offsets at 406-431), optional per-pixel rotation (filterMode>=1, hash of uv texel+depth), **PCSS** blocker search (filterMode>=2, non-compare path only, 318-355); cascade blend band mixing next cascade (547-573); translucent shadow = 3 RGB moment maps, exponential transmittance with erf-CDF (201-219, 439-469). Final: `light *= shadowInfo.x * translucent` before the same diffuse/specular composition (784-820). 15 debug modes.

`shadow_point_interaction.vs/.fs`: adds `uModelMatrixRow0..2` (world transform rows), `uGlobalLightOrigin`, `vPointShadowVector = worldPos - lightOrigin + worldNormal*normalOffset*dist*sinTheta` (65-73); FS: `depth = length(vector)/uPointShadowFar`, direction into cube; compare via `samplerCubeShadow` (OPENQ4_POINT_SHADOW_COMPARE) or color-encoded depth (`uPointShadowDepthMode`: fp16 .r vs RG 16-bit packed `rg.x + rg.y/255`, 206-215); PCF = tangent-frame disc offsets around the direction scaled by `uPointShadowTexelScale = 2/faceWidth` (235-292).

**Receiver pass CPU side** (units differ from ARB2!): GLSL paths bind bump=0, falloff=1, lightProj=2, diffuse=3, specular=4, shadow map=5, translucent moments=6/7/8 (9834-9971 projected, 10101-10220 point). Projected receiver per-space upload: cascade clip planes localized per `surf->space->modelMatrix` (`R_GlobalPlaneToLocal`), cached via `g_shadowMapInteractionLastSpace` (8549-8582). Atlas rect composed at upload: `slotOrigin + stateRect * slotSpan` in atlas UV (9899-9918). Depth image bound with COMPARE_R_TO_TEXTURE/LEQUAL + LINEAR when `RB_ShadowMapDepthCompareEnabled` (9962-9971). Per-draw uniforms mirror drawInteraction_t exactly like ARB2 (8584-8615). State: same `GLS_SRCBLEND_ONE|GLS_DSTBLEND_ONE|GLS_DEPTHMASK|depthFunc`, stencil forced ALWAYS 128 (9981-9983). Per-light filter settings from `R_ShadowMapProjectedFilterSettings(vLight)` (9832).

## 4. Phase F deferral candidates (my assessment — marked speculation where judgement)

Safe to defer (all cvar-gated or fallback-covered in GL):
- **Translucent interactions**: separate chain, drawn after both passes with depthFunc LESS; deferring loses per-light lighting on translucents only. Stencil-shadowed translucents (`r_stencilTranslucentShadows`) and shadow-map translucent receivers (`r_shadowMapTranslucentReceivers`) are additional layers on top.
- **Translucent shadow moment maps** (VSM RGB, `RB_RenderTranslucentShadowMap`/`RB_RenderPointTranslucentShadowMap` 8161/8283, r_shadowMapTranslucent*): fully optional (`uTranslucentShadowEnabled` uniform defaults off path).
- **PCSS** (filterMode>=2) and rotated-Poisson (filterMode>=1): keep plain 5-tap PCF first.
- **CSM cascades**: state struct always carries 4 (`SHADOWMAP_MAX_CASCADES`); can force cascadeCount=1 initially (r_shadowMapCSM/r_shadowMapCascadeCount, defaults 4-cascade when CSM on).
- **Static atlas caching + compose-dynamics + admission scheduling** (`RB_ShadowMapSchedulePass`, cache entries, `glBlitFramebuffer` compose 7852-7864): render every mapped light to scratch every frame first; the persistent atlas (r_shadowMapAtlasSize 4096, cells of r_shadowMapSize 1024, CSM 2x2 blocks) is an optimization.
- **MD5R packed vertex path** (`_MD5R_SUPPORT` blocks): only needed if renderer-vk consumes packed prim-batch meshes; the classic ambientCache path covers everything else.
- **Custom GLSL material interaction stages** (9437-9700) and the wrapped-custom-GLSL receiver (8863): defer; GL falls back to filtered stencil for these surfaces.
- **Debug modes/overlay, GPU timers, stats reporting, Apple GL2.1 corridor, SimpleInteraction rescue family**: GL-specific resilience machinery, not contract.
- **Specular table**: the GLSL parity path already replaces the table lookup with the analytic `(4h·n-3)²` term, so Vulkan can go analytic (speculation: retail .vfp table and the analytic term match closely; this repo already shipped the analytic version as the stock GLSL path).
- **Light cull sides**: there is no per-light side culling in the ARB2 consumer — only per-surface scissor + stencil; caster-vs-cascade/face rejection (`RB_ShadowMapCasterOutsideCascade` 7067, `RB_PointShadowMapCasterOutsideFace` 7547) is a perf cull, deferrable.

NOT deferrable for parity: the two-pass local/global ordering, per-light-stage × per-surface-stage interaction decomposition, blackImage/flatNormal substitutions and the skip-if-black rule, vertexColor modes, specularColor×2, lightScale, MF_POLYGONOFFSET, depth hacks, per-surface scissor.

## 5. r_useShadowMap in the interaction draw + caster pass

**How the interaction draw changes**: under a mapped light the ARB2 program is NOT used for eligible receivers — the whole interaction chain is redrawn by dedicated GLSL receiver programs (`shadow_interaction` / `shadow_point_interaction`) that fold the shadow term into the same additive interaction math (mask pass = lit pass). Ineligible surfaces (custom GLSL lighting w/o wrapper, GPU-generated/posed geometry, invalid, prepare-failed — reasons at 8896-8931, stage filter `RB_ShadowMapReceiverStageFilter` 8877) get per-surface filtered fallbacks: `RB_ShadowMapDrawReceiverFallbacks` (10813) and `RB_ShadowMapStencilFallbackFiltered` (10393: clears stencil in light scissor, replays stencil volumes, then ARB2 interactions for only the filtered surfaces — avoids double-lighting).

**RB_ShadowMapRunPass decision tree** (10855-11079), per light per pass: no casters & no stencil surfs → unshadowed `RB_DrawMaterialInteractions` (10872-10889); casters absent but stencil surfs exist → full stencil fallback (10891-10905); no eligible receivers → receiver fallbacks only (10907-10911); schedule REUSE → optional compose-dynamics then receiver mask from cached map (10921-10983); RENDER → `RB_RenderShadowMap`/`RB_RenderPointShadowMap` → optional translucent moment render → receiver mask → fallback fills; any failure → sticky stencil fallback (`RB_ShadowMapMarkStencilFallbackSticky` restores stencil volume generation next frame).

**Where shadow maps are rendered**: mid-frame, inside the per-light loop, between the previous light and its own receiver pass (NOT a frame-level prepass). `RB_RenderShadowMap` (7795-7984) state requirements: switch FBO to scratch or persistent-atlas depth render texture (`g_shadowMapRenderTexture`, depth-only via `g_shadowMapDepthImage`; selection 3259-3270/5691-5801); completeness probe fails the pass gracefully (7839-7850); per cascade: viewport+scissor = tile (tileX/Y from slot origin + cascade%atlasDiv * tileSize, 7899-7904), `glClear(GL_DEPTH_BUFFER_BIT)` per tile (skipped in compose mode), projection = `R_ShadowMapClipPlanesToGLMatrix(state.clipPlanes[cascade])` (7902-7911); caster program for ALL casters with **shader-written depth**: `gl_FragDepth = clamp(vShadowDepth + offset.x*max(|dFdx|,|dFdy|) + offset.y, 0, 1)` (`shadow_proj_caster.fs:68-69`) because glPolygonOffset doesn't affect shader depth; `vShadowDepth = dot(position, uShadowDepthRow)` per cascade, depth<=0 (apex side) discarded, past-falloff clamps to far (fs:56-63); alpha-tested/hashed-alpha perforated casters sample the material alpha map on unit 0; fixed state: colorMask off, depth LEQUAL + write on, no blend, no stencil, scissor on (7885-7892). Full view-state restore after (viewport/scissor/projection/blend/stencil/cull/tmu invalidate, 7930-7976). Returns false if an all-skipped opaque chain drew nothing (7983).

**Point lights** `RB_RenderPointShadowMap` (7986-8159): 6 cube faces, per-face `MakeCurrent(face)`, clear color (1,1,1,1) + depth, shared perspective projection (near = clamp(far*0.01, 0.5, 16), far = `RB_PointShadowMapLightFar`), per-face view matrix from light origin; **colorMask ON** (radial depth is color-encoded unless depth-compare mode), DEPTH_CLAMP when available; state restore identical.

**Cascade/atlas state** consumed by the receiver: `shadowMapProjectedLightState_t` (ShadowMapProjected.h:40-62): `valid, cascadeCount, atlasDiv, tileSize, clipPlanes[4][4] (global-space rows; localized per entity at draw), atlasRect[4] (per-light convention; composed with slot origin at upload), splitDepths[4], biasScale[4], texelDepthBias[4], worldTexelSize[4]`; built by `RB_ShadowMapBuildProjectedState` (4369) via `R_BuildShadowMapProjectedLightState`. `RB_ShadowMapBuildArb2ParityState` (3206) + `ShadowMapArb2Parity.h` expose this contract for cross-checking (parallel lights ride the projected path with a synthesized ortho projection, 6018-6020). Atlas-slot lookup for external consumers: `RB_ShadowMapProjectedAtlasSlotForLight` (9383). Cvars: r_shadowMapSize 1024, r_shadowMapAtlasSize 4096, r_shadowMapPointSize 512, r_shadowMapPointLights 1 (RenderSystem_init.cpp:288,315-317,525,533-543).
