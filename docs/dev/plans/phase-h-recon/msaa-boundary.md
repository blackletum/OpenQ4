# Phase H recon — MSAA slice + H/I boundary (ground truth)

## 1. MSAA in the GL backend

**Cvars / normalization**
- `r_multiSamples` default **"0"**, CVAR_ARCHIVE|INTEGER, range 0–16, discrete completion {0,2,4,8,16} — `src\renderer\RenderSystem_init.cpp:212,217`. Normalized to those steps pre-GL-init by `R_NormalizeMultiSamplesValue` (`RenderSystem_init.cpp:1504-1518`) from `R_NormalizeDisplayCvars` (`:1561-1567`). Token-pinned by `tools\tests\renderer_msaa_cvar_safety.py` (asserts exact source strings in RenderSystem_init.cpp and the openQ4-game companion — keep wording intact or update the test).
- `r_msaaAlphaToCoverage` default "1" (`RenderSystem_init.cpp:271`); `r_msaaResolveDepth` default "0" (`:270`, sole reader is game `openQ4-game\src\game\Game_render.cpp:1264`).

**Context creation (window-surface MSAA)** — `parms.multiSamples = r_multiSamples` (`RenderSystem_init.cpp:1633`; safe-mode retry resets to 0 at `:1651`). SDL3 ladder tries each context candidate × sample-fallback list, then writes the achieved count back into `r_multiSamples` (`src\renderer\OpenGL\gl_ContextSDL3.cpp:354-431`, write-back `:425-429`). This only affects the default framebuffer; the real scene MSAA is FBO-based.

**Two independent MSAA render-target consumers**
1. Engine FP16 scene target: `RB_EnsureSceneRenderTexture` (`draw_common.cpp:1603-1662`) allocates `_hdrSceneColor` RGBA16F + `_hdrSceneDepthStencil` with `numMSAASamples = r_multiSamples`, but only when **not** supersampling and **not** modern-visible post (`:1613-1616`). MSAA scene color can't be presented directly (`canPresentSceneColorDirectly` requires ≤1 sample, `draw_common.cpp:3781-3785`) so present resolves via `RB_CaptureCurrentRenderImage`/`CopyFramebuffer` blit.
2. Game openQ4 post chain: `openQ4_CreateForwardRenderTarget` (`openQ4-game\src\game\Game_render.cpp:933-1002`) allocates `_forwardRenderAlbedo` RGBA8 + `_forwardRenderDepth` at `r_multiSamples` with a fallback ladder (`openQ4_NextLowerMSAASampleCount` `:920-931`) that depends on `GetImageMSAASamples` (`RenderSystem.cpp:2090-2092`, returns `opts.numMSAASamples`) reflecting the **allocated** count — the GL path clamps `opts.numMSAASamples` in place (`OpenGL\gl_Image.cpp:493-507`). Then `renderSystem->ResolveMSAA(forwardRenderPassRT, forwardRenderPassResolvedRT, blurEnabled || r_msaaResolveDepth)` at `Game_render.cpp:1261-1264` — this is the **only** ResolveMSAA caller anywhere.

**RC_RESOLVE_MSAA** — front-end `idRenderSystemLocal::ResolveMSAA` (`RenderSystem.cpp:1864-1879`); GL executor `RB_ResolveMSAA` (`tr_backend.cpp:630-685`): per-color-attachment `glBlitFramebuffer` (COLOR, NEAREST), optional depth blit when `cmd->resolveDepth`, then restores the tracked render texture (`RB_RestoreTrackedRenderTexture` `:616-623`).

**Alpha-to-coverage** — `RB_UseAlphaToCoverage` (`draw_common.cpp:623-646`): needs cvar + MC_PERFORATED + ARB_multisample + `backEnd.renderTexture` color image with >1 samples. Applied in depth fill (`:5954`, `glEnable(GL_SAMPLE_ALPHA_TO_COVERAGE)` `:5975-5977`) and three other stage walks (`:5432`, `:6543`, `:8750`).

**Effective-MSAA gating** (`R_GfxInfoPrintAAState`, `RenderSystem_init.cpp:3422-3500`): suppressed by supersampling (screenFraction>100), modern-visible post, missing ARB_texture_multisample, and GL_MAX_SAMPLES clamp. MSAA texture alloc = `GL_TEXTURE_2D_MULTISAMPLE`/`glTexImage2DMultisample` (`gl_Image.cpp:463-507`); FBO attach `gl_RenderTexture.cpp:430-441`.

**What vk MSAA needs**
- Swapchain images are 1-sample (`Vulkan\VulkanDevice.cpp:279`, usage COLOR_ATTACHMENT|TRANSFER_DST); per-slot depth is 1-sample (`:145-153`). MSAA ⇒ offscreen multisampled color (swapchainFormat) + depth (vkCtx.depthFormat) transients, resolved via `VkRenderingAttachmentInfo::resolveImageView` (dynamic rendering already in use) or `vkCmdResolveImage`.
- Every pipeline goes through shared `VK_Exec_CreatePipeline` which hardwires `rasterizationSamples = VK_SAMPLE_COUNT_1_BIT` (`vk_GuiExecutor.cpp:332-335`, bound at `:402`); caches key **only on blendBits** (`VK_GuiExecutor_GetPipeline` `:416-427`; cube variant `:469+`; interaction/shadow pipelines in `vk_Interactions.cpp`/`vk_ShadowMap.cpp` use the same helper). Pipeline-cache implication: key must gain sample count (or flush all pipelines on sample change), plus an `alphaToCoverageEnable` bit for perforated depth-fill/ambient parity with `draw_common.cpp:5954/5975`. Shadow atlas stays 1-sample (`vk_ShadowMap.cpp:265,375`).
- `vk_Image.cpp` AllocImage **ignores** `opts.numMSAASamples` (samples hardwired at `vk_Image.cpp:377`) and does not zero it — so `GetImageMSAASamples` lies under vk: the game ladder (`Game_render.cpp:963-982`) would believe MSAA was allocated. Phase H must either honor `numMSAASamples` in vk AllocImage (color/depth-attachment usage + resolve pairing) or clamp `opts.numMSAASamples = 0` so the game path stays truthful.
- Clamp against `VkPhysicalDeviceLimits::framebufferColor/DepthSampleCounts` (analog of the GL_MAX_SAMPLES clamp `gl_Image.cpp:495-502`).

**Is MSAA pure opt-in?** Cvar default 0, yes — **but** `Com_ExecMachineSpec_f` sets r_multiSamples 8/4/2 for machineSpec 3/2/1 (`framework\Common.cpp:2280,2305,2327`; 0 only for low `:2351`), and the performance-preset system applies/verifies it (`Common.cpp:2716`, high tiers require ≥2/≥4 `:3148,:3157`). Settings menu also exposes it (`docs\dev\settings-menu-registry.json`). So preset users on high/ultra get MSAA silently dropped under vk until H lands — not a correctness break, but a visible-quality parity gap.

## 2. r_screenFraction / supersampling

- Defs: `r_screenFraction` default "100", range 10–200 (`RenderSystem_init.cpp:567`); `r_resolutionScaleMode` default "1" (0 legacy crop / 1 bilinear / 2 HQ upscale, `:568`); `r_resolutionScaleSharpness` (`:569`).
- Readers:
  - **<100, mode 0**: front-end crop in BeginFrame (`RenderSystem.cpp:1370-1379`, `CropRenderSize`) — API-agnostic, works under vk today.
  - **<100, modes 1/2**: GL swap-time fullscreen pass `RB_ApplyResolutionScaleToBackBuffer` (`draw_common.cpp:3916-4017`) called from `RB_SwapBuffers` (`tr_backend.cpp:553`), samples the back buffer on a reduced grid via GLSL (`:3969-4013`). GL-only TU — vk has no equivalent (H post family, alongside the CRT/color-mapping swap passes `RB_ApplyCRTToBackBuffer` `draw_common.cpp:4064`, `RB_ApplyColorMappingsToBackBuffer` `:4292`, both invoked `tr_backend.cpp:554-555`).
  - **>100 supersampling**: scene-target path — `RB_ComputeSupersampledSceneSize` (`draw_common.cpp:1097-1111`, clamped by GL max texture size `:1042-1077`), `RB_BeginSceneSupersampling` (`:1190-1239`, rescales viewDef viewport/scissor plus every drawSurf/light/entity scissor in place), restore `:1241-1249`, downsample happens in `RB_PresentSceneRenderTargetToBackBuffer` (`:3760-3877`). Supersampling deliberately forces the scene target to 1-sample (no MSAA stacking, `:1613-1616`). Requesting the scene target for supersampling: `RB_ViewRequestsSceneRenderTarget` (`:1310-1342`).
- Perf presets pin r_screenFraction (100 for mid+/85 for low, `Common.cpp:2282,2307,2329,2353`; preset checks `:3121-3165`).

## 4. RenderDoc integration

The module-side forwarder `RenderDoc_IsInjected` lives in `RendererGLModule.cpp:132-134`, which **is compiled into renderer-vk** (not in `RENDERER_VK_EXCLUDED_SOURCES`, `tools\build\meson_sources.py:112-126`). Its only renderer consumer is the GL-failure path `R_ErrorForMissingRequiredOpenGLFeatures` (`RenderSystem_init.cpp:102-108`), which is GL-extension-check specific. **Conclusion: the vk module needs nothing new for Phase H.** `tools\debug\renderdoc_capture.ps1` already parameterizes `+set r_multiSamples` (line 82).
