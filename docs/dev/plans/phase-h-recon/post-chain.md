# Phase H recon — openQ4 post chain (bloom/SMAA/motion blur/SSAO) + tone mapping + gamma + scene RT + RT infra

## 1. Chain enumeration and ordering

### Engine backend chain (tail of RB_STD_DrawView, `src/renderer/draw_common.cpp:9821-9848`)
Order after fog/blend + post-fog material passes:
1. `RB_CaptureSceneRenderTargetPreserveDepthImage()` (9821, def 1574) — depth snapshot for depth-aware present
2. `RB_STD_SSAO()` (9824, def 2068)
3. `RB_STD_MotionBlur()` (9827, def 2725)
4. `RB_STD_Bloom()` (9830, def 3351) — this single pass is ALSO the tone-map/color-grade/HDR-debug composite
5. Authored SS_POST_PROCESS surfaces: `RB_STD_DrawShaderPasses(drawSurfs+processed, ...)` (9833-9835; `processed` from `RB_STD_FindPostProcessStart` 9755)
6. `RB_RenderDebugTools` (9837)
7. `RB_PresentSceneRenderTargetToBackBuffer` (9839-9841, def 3760) if root scene RT active; `RB_RestoreSceneSupersampling` (9842)

### Swap-time back-buffer chain (`src/renderer/tr_backend.cpp:539-565`, RB_SwapBuffers)
`RB_ApplyResolutionScaleToBackBuffer` (553; draw_common.cpp:3916 — only r_screenFraction<100 && r_resolutionScaleMode 1/2) → `RB_ApplyCRTToBackBuffer` (554; def 4064 — only r_crt) → `RB_ApplyColorMappingsToBackBuffer` (555; def 4292 — r_gamma/r_brightness) → `GLimp_SwapBuffers` unless `tr.takingScreenshot` (562-564). These run AFTER 2D/GUI, i.e. they touch every final pixel.

### SSAO (`RB_STD_SSAO` draw_common.cpp:2068-2243)
- Gate `RB_SSAORequestedForCurrentView` (2000-2011): !r_skipPostProcess && r_ssao && GLSL && main-scene view (`RB_IsMainScenePostProcessView` 914-954: needs viewEntitys, not portal-sky/subview/superView/subviewSurface/negative viewID/xray) && radius>0 && intensity>0.
- Inputs: `_currentRender` via CopyFramebuffer (2098); depth = world-only depth `rbSSAOWorldDepthImage` captured pre-transparency by `RB_CaptureSSAOWorldDepthImage` (6312, invoked at 9707; conditional capture also at 5927) else `_currentDepth` via `RB_CaptureCurrentDepthImage` (1534); plus `_ssaoFinalDepth` CopyDepthbuffer (2120-2129).
- One fullscreen quad with `ssao.fs` (name set 2038) blended over the current target in place; uniforms invTexSize/projectionInfo/depthProjection/projectionScale/radius/bias/intensity/power/maxDistance/sampleCount/debug (2176-2231). Helpers: `RB_BeginFullscreenPostProcessPass` 1681, `RB_DrawFullscreenPostProcessQuad` 1724, `RB_EndFullscreenPostProcessPass` 1838.

### Motion blur (`RB_STD_MotionBlur` draw_common.cpp:2725-2905)
- Gates 2726-2747: r_motionBlur && GLSL; main motion-blur view only (1255-1266: main scene view, no subview/superView, viewID>=0); r_jitter off; strength/maxPixels/samples > 0. Resets history on any failure.
- Previous-frame view state (`rbMotionBlurViewState_t` 2264-2280) validated by `RB_MotionBlurHistoryUsable`; object vectors when r_motionBlurObjectVectors (default 1): renders a motion-vector buffer into its own idRenderTexture `rbMotionVectorRenderTexture` (2301-2302) via `RB_RenderMotionVectorBuffer` (2798) with `motionvectors.fs`; entity matrix history lists 2288-2300.
- Captures `_currentRender` + `_currentDepth` (2792-2795), then one fullscreen `motionblur.fs` pass (samplers scene/depth/motionvector 2818-2827; uniforms reconstruct/project info, view axes prev+curr, params 2838-2889). Sets `backEnd.currentRenderCopied = false` after (2904).

### Bloom + tone map composite (`RB_STD_Bloom` draw_common.cpp:3351-3636)
- Runs if bloomRequested (`r_bloom && r_bloomIntensity>0.0001`, 1251) OR toneMapEnabled (`r_hdrToneMap`) OR hdrDebugView>0 (3356-3361); needs GLSL + main scene view.
- Bloom pyramid (only when bloomRequested): up to 5 levels (`RB_BLOOM_MAX_LEVELS`=5 at 960; count r_bloomMipCount), per level into FP16 ping-pong RTs `rbBloomRenderTextures[level][0/1]` (2961-2962; FMT_RGBA16F at 3100/3165): level 0 = `bloom_extract.fs` (threshold+softKnee), else `bloom_downsample.fs`; then `bloom_blur.fs` horizontal into [1] and vertical back into [0] with per-level radius `r_bloomRadius*(1+level*0.65)` (3439-3533). Weights 0.34/0.24/0.17/0.14/0.11 normalized (962-964, 3531-3532).
- Composite `bloom.fs`: scene + 5 bloom textures + uniforms bloomIntensity/bloomEnabled/toneMapEnabled/hdrExposure/whitePoint/lift/postGamma/gain/vibrance/saturation/contrast/highlightDesaturation/gamutCompression/debugView/weights (2932-2953, 3554-3616). Exposure = r_hdrExposure × auto-exposure from `RB_UpdateHDRAutoExposure` (3199) only when `RB_HDRAutoExposureEnabled` (1306: r_hdrAutoExposure && r_hdrToneMap && ModernVisiblePostProcessHandoffActive — i.e. modern-GL only). Auto-exposure uses `hdr_luminance.fs` pyramid + double-buffered PBO async readback (997-1001, r_hdrAutoExposureAsync).
- Output: to `originalRenderTexture` (scene RT) or backbuffer; `RB_SetFramebufferSRGBEnabled(true)` around the backbuffer draw is currently hard-neutered (`strictLinearOutputEnabled = false` at 1500, draw_common.cpp:1495-1510 — GL_FRAMEBUFFER_SRGB never enabled). Re-captures `_currentRender` when RT (3633-3635) so SS_POST_PROCESS materials sample the bloomed scene.

### SMAA — NOT an engine backend pass; game-DLL-driven material passes
- Engine holds only: cvar `r_postAA` 0-4 (RenderSystem_init.cpp:218), lookup images `_smaaArea`/`_smaaSearch` (Image_intrinsic.cpp:705-706, validated RenderSystem.cpp:2029-2050), the RC_SET_POSTPROCESS_* command family (tr_local.h:414-416 → backEnd fields, tr_backend.cpp:878-886), material shaderParm plumbing `postProcessSMAAQuality` etc. (Material.cpp:1511, Material.h:254, draw_common.cpp:542-543), and an explicit-fullscreen-quad path for SMAA GLSL stages (`RB_IsSMAAPostAAGLSLProgram` draw_common.cpp:1755-1766, `RB_DrawSMAAExplicitFullscreenQuad` 1785, used in shader-pass dispatch 6626/6695-6697).
- Driver: `E:/Repositories/openQ4-game/src/game/Game_render.cpp`. `idGameLocal::RenderScene` (1128-1300) reads r_postAA via `BuildPostAAPasses` (1201). `openQ4_ApplySMAA` (792-855) runs 4 passes, each = optional `SetPostProcessSourceColorSpace/SourceSize` → `BindRenderTexture(output, NULL)` → `ClearRenderTarget` → fullscreen `DrawStretchPic`:
  1. edge: material `postprocess/smaa_edge` (smaa_edge.fs, reads `_postProcessAlbedo2` = scene source) → `_postProcessAlbedo1`
  2. weights: `postprocess/smaa_weights` (smaa_weights.fs, reads `_postProcessAlbedo1` + `_smaaArea` + `_smaaSearch`) → `_postProcessAlbedo0`
  3. blend: `postprocess/smaa_blend` (reads `_postProcessAlbedo2` + `_postProcessAlbedo0`) → `_postProcessAlbedo1`
  4. normalize: `postprocess/copy_postprocess1` → `_postProcessAlbedo0`
  Materials in `content/baseoq4/pak0/materials/postprocess_openq4.mtr` (smaa_edge at ~line 54, weights ~68, blend ~84); all `sort postProcess`. Quality presets → `SetPostProcessSMAAQuality` idVec4 (x=edge mode, y=threshold, z=search steps, w=local contrast; tr_local.h:633).
- Full game frame when post path active (Game_render.cpp:1252-1298): bind `forwardRenderPassRT` (MSAA, FMT_RGBA8 — deliberately LDR, openQ4_CreateForwardRenderTarget 933-942 with sample-count fallback ladder 919-930) → clear → world render → `ResolveMSAA(forwardRenderPassRT, forwardRenderPassResolvedRT, blur||r_msaaResolveDepth)` (1261) → resolve pass draws `postprocess/resolvepostprocess` (samples `_forwardRenderResolvedAlbedo`) into postProcessRT → SMAA → final fullscreen material (`nopostprocess`/`casupscale`/`blur`, openQ4_SelectFinalPostProcessMaterial 854-862) to backbuffer with a depth clear (1289).
- MSAA source: `r_multiSamples` read at Game_render.cpp:1018; engine GL resolve = `RB_ResolveMSAA` glBlitFramebuffer per color attachment + optional depth (tr_backend.cpp:630-685).

### DEFAULTS — what the retail-default vk path actually needs
Out-of-box cvar defaults (RenderSystem_init.cpp): r_postAA **0** (218), r_bloom **0** (220), r_ssao **0** (226), r_motionBlur **0** (234), r_hdrToneMap **0** (243), r_hdrAutoExposure 0 (252), r_crt 0 (264), r_multiSamples **0** (217), r_screenFraction 100 (567), r_gamma 1 / r_brightness 1 (413-414), r_skipPostProcess 0 (462). r_hdrSceneTarget defaults **1** (242) but only matters with modern-visible GL (`RB_ModernVisibleSceneTargetRequested` 1298-1300), which is off (r_rendererModernVisible 0 at 440, r_rendererModernAutoPromote 0 at 441, evidence-gated RendererBootstrap.cpp:322-349). Shipped `openq4_defaults.cfg` is binds only; no cfg flips post cvars.
- Game defaults: g_renderFastNoPost **1**, g_renderFastNoPostDirect **1**, g_renderCaptureCurrentRender 0, g_renderCasUpscale 0 (Game_render.cpp:8-11). So at pure defaults the game takes `openQ4_RenderSceneDirect` → backbuffer; the ENTIRE engine post chain and the game RT path are no-ops. Zero post passes needed for the strict out-of-box frame.
- BUT presets: `applyPerformancePreset`/`autoDetectPerformancePreset` (menu-driven, Session_menu.cpp:2792-2806) — table Common.cpp:2446-2476: performance/balanced/quality/ultra all set **r_postAA=1** and multiSamples **0/2/4/8**; default preset name "balanced" (2479). ALL presets keep bloom/ssao/hdrToneMap/motionBlur/crt at 0 (fields 17-21 of each row; self-test asserts this, Common.cpp:3112). Legacy menu "video quality" path (`execMachineSpec`, Session_menu.cpp:2843) sets r_postAA=1 + multiSamples 2/4/8 for spec 1/2/3 (Common.cpp:2281-2352). So the practical near-default target is **MSAA 2x + SMAA 1x medium via the game RT path**; bloom/SSAO/motionBlur/tonemap/CRT/supersampling are opt-in.

## 2. Port priority per piece
- **Priority (preset-reachable): game-driven RT path** = RC_SET_RENDERTEXTURE + RC_CLEAR_RENDERTARGET + RC_RESOLVE_MSAA + RC_COPY_RENDER + RC_SET_POSTPROCESS_* + MSAA render targets + SS_POST_PROCESS material stages with GLSL programs (smaa_*.fs) + the explicit fullscreen quad rule for SMAA stages. Sequence exactly as in Game_render.cpp:1252-1298 above.
- **Opt-in scope** (engine chain, draw_common.cpp is NOT compiled into the vk module — all need native vk reimplementation): SSAO, motion blur (incl. motion-vector RT), bloom pyramid + tonemap composite, HDR auto-exposure readback, CRT, resolution-scale upscale, supersampling (screenFraction>100), scene FP16 RT + depth-aware present. Reference math test: `tools/tests/hdr_postprocess_math.py` (tone map, lift/gamma/gain, display color mapping, legacy gamma table at lines 138-166). Shaders all in `content/baseoq4/pak0/glprogs/` (bloom*.fs, ssao.fs, motionblur.fs, motionvectors.fs, hdr_luminance.fs, crt.fs, resolutionscale.fs, smaa_*.fs, blur.fs).

## 3. Gamma/brightness path — affects every pixel
- `R_SetColorMappings` (RenderSystem_init.cpp:3359-3389) builds `tr.gammaTable` then calls `GLimp_SetGamma` — which is a **no-op on every backend**: win32 body commented out (src/sys/win32/win_glimp.cpp:190-207), SDL3 stub (src/renderer/OpenGL/gl_ContextSDL3.cpp:173-177), vk stub (src/renderer/Vulkan/vk_Backend.cpp:294-298). `GLimp_UseNativeGammaRamps` returns false everywhere (win 209, SDL3 179, vk 300).
- Real mechanism: **shader pass at swap** `RB_ApplyColorMappingsToBackBuffer` (draw_common.cpp:4292-4358), gated on non-neutral values (4287-4290: |brightness-1|>1e-4 or |gamma-1|>1e-4). Copies backbuffer to `_currentRender` then fullscreen quad with inline GLSL: `color = pow(clamp(rgb*brightness,0,1), 1/max(gamma,0.001))` (4205-4216). No `r_gammaInShader` cvar exists — this is unconditional when non-neutral.
- vk requirement: an equivalent final swapchain-image pass (post-GUI) with the identical formula. At defaults (1/1) it must be skipped — so deterministic capture gates are unaffected out of the box; parity math reference is `apply_display_color_mapping` in hdr_postprocess_math.py:145-148. Note screenshots read the back buffer AFTER this pass (tr_backend.cpp:558-564), so the vk readback must capture post-gamma output for parity.

## 4. Scene-render-target path (_hdrScene*, engine-side)
- `RB_ViewRequestsSceneRenderTarget` (draw_common.cpp:1310-1342): !r_skipPostProcess && main-scene view && GLSL, then ANY of: bloom, motion blur, ssao, r_hdrToneMap, auto-exposure, hdrDebugView>0, modern-visible+r_hdrSceneTarget, supersampled (screenFraction>100). `RB_SceneRenderTargetRequested` 1664, inline-subview variant 1671, portal-sky routing 1412+.
- Creates `_hdrSceneColor` FMT_RGBA16F + `_hdrSceneDepthStencil` RT (`RB_EnsureSceneRenderTexture` 1603-1662); the RT itself carries r_multiSamples MSAA when not supersampled/modern (1613-1616) — the engine's own MSAA story for the opt-in chain. Bound at 9691-9701 (`backEnd.renderTexture->MakeCurrent()`), presented by `RB_PresentSceneRenderTargetToBackBuffer` (3760) using a depth-aware present program (3662+) with preserved far depth for portal-sky (1439-1600).
- `_forwardRenderResolved{Albedo,Depth}` are intrinsic images (Image_intrinsic.cpp:709-710) written by the GAME's resolve pass and by `EmitFullscreenSpecialEffects` (RenderSystem.cpp:741-742) — distinct from `_hdrScene*`.
- Reachability under vk pins: at strict defaults **unreachable** (all triggers off, modern-visible off). Becomes live the instant any of bloom/ssao/toneMap/motionBlur/screenFraction>100 is set — but the code is GL-only (draw_common.cpp excluded from vk build); vk needs native equivalents, not this code path.

## 5. Render-texture infra under vk
- idRenderTexture GL impl: `src/renderer/OpenGL/gl_RenderTexture.cpp`; class `src/renderer/RenderTexture.h:45`. vk stubs: vk_Backend.cpp:410-460 (`MakeCurrent` returns false, `EnsureDeviceHandle`/`GetDeviceHandle` inert).
- Emitters: `BindRenderTexture` → RC_SET_RENDERTEXTURE (RenderSystem.cpp:1819-1836, flushes guiModel first), `ClearRenderTarget` → RC_CLEAR_RENDERTARGET (1843-1857), `ResolveMSAA` → RC_RESOLVE_MSAA (1864-1879), `CaptureRenderToImage`/`CaptureDepthRenderToImage` → RC_COPY_RENDER (1677-1714, 670-711). GL consumers: RB_SetRenderTexture tr_backend.cpp:600-614, RB_ClearRenderTarget 692-721, RB_ResolveMSAA 630-685, RB_CopyRender 574-592. vk `RB_ExecuteBackEndCommands` (vk_Backend.cpp:336-387) handles only RC_SET_BUFFER/RC_DRAW_VIEW/RC_SWAP_BUFFERS; everything else hits `default:` and is dropped.
- What breaks while dropped: (a) r_postAA>0 or r_multiSamples>1 or weapon-wheel blur (SPECIAL_EFFECT_BLUR toggled from openQ4-game Player.cpp) or g_renderCasUpscale → the game's full path emits binds/clears/resolves that vanish, and the final fullscreen material (`postprocess/nopostprocess` sampling never-written `_postProcessAlbedo0`) paints stale/black over the scene — the frame is destroyed, not just missing AA; (b) `EmitFullscreenSpecialEffects` blur capture (RenderSystem.cpp:729-757: RC_COPY_RENDER of `_postProcessAlbedo0`+`_forwardRenderResolvedDepth`) broken; (c) all `_currentRender` feedback (heat haze, postprocess-sorted materials) and security-camera/remote-screen subview captures rely on the RC_COPY_RENDER family (other slice's detail); (d) SMAA lookup textures exist but are never sampled. Note the game marks its scene RT as its own feedback target (`openQ4_BindSceneRenderTexture` Game_render.cpp:729-736) so `_currentRender` on-demand copies come from the active FBO — the vk RT implementation must support the renderTexture+feedbackRenderTexture pair semantics of setRenderTargetCommand_t.
- ScenePackets front-end capture mirrors the same request logic (bloom 1580, motionBlur 1586-1594, ssao 1608, RC_SET_POSTPROCESS_SMAA_QUALITY 1734 in src/renderer/ScenePackets.cpp) — reusable gating reference for vk scheduling.
