# openQ4 0.6.5 Release Notes

## Highlights

- Runtime packaging is more robust: openQ4 PK4s now track `content/baseoq4/pak0/` and `content/baseoq4/pak1/` through generated source manifests, so removed generated content no longer leaves stale build dependencies.
- Release packaging is unblocked when Apple signing credentials are not configured. The release workflow keeps Windows/Linux artifacts mandatory, publishes signed/notarized experimental macOS DMGs when the Apple Developer ID and notary secrets are present, and otherwise publishes clearly labeled unsigned/unnotarized experimental macOS archives.
- The renderer does less repeated CPU work in GL, material, submit, and AAS-heavy paths by caching stable loop counts, stage metadata, material masks, and texture binding state.
- Windows windowed play defaults to a smoother borderless presentation path, reducing OpenGL swap stalls seen in BSE-heavy outdoor scenes such as Air Defense 1.
- Sound playback is closer to retail Quake 4, including attenuation, master volume, frequency-shift behavior, restart/reload diagnostics, and cinematic skip timing.
- OpenAL audio has been restored to the end-of-day June 23 sound implementation after reliability reports on the newer OpenAL work. The temporary experimental voice controls have been removed instead of shipping as user-facing compatibility switches.
- Performance presets are fully wired through the System settings menu and console commands, with conservative memory telemetry handling, shared menu/console command validation, registry coverage, a validation self-test, low-power sound-shader sample caps, and corrected audio restoration when moving back from constrained profiles.
- Multiplayer joins now open on a Quake Live-like entry screen with server details and gametype-aware join choices, while the client waits as a spectator at the map intermission view when possible.
- Experimental macOS push verification builds are healthy again after replacing raw C string compares in process handoff and environment filtering with openQ4's guarded string comparison API.

## Upgrade Notes

- This release publishes Windows x64, Windows ARM64, Linux x64, Linux ARM64, and experimental macOS ARM64 packages, including Windows installers, Linux debug-symbol archives, and separate macOS dSYM symbol archives for crash symbolication.
- Release packages now use optimized binaries on Windows, Linux, and experimental macOS while keeping Windows PDBs and Linux debug-symbol archives available for crash reports.
- If Apple signing and notarization secrets are not configured, the experimental macOS OpenGL and Metal bridge downloads are named `-unsigned.tar.gz`, are ad-hoc signed only for bundle validity, and may require normal Gatekeeper approval on first launch. Credentialed release runs publish signed/notarized DMGs instead.
- Set `ui_autoJoin 1` before connecting if you prefer the previous multiplayer behavior that immediately enters the match instead of showing the join screen.
- OpenAL now uses the June 23 sound implementation directly. Remove any local experimental audio overrides from launch configs; they are no longer supported settings.
- Replace an existing openQ4 package with the matching new archive or installer, keeping your retail Quake 4 `q4base/` assets separate from openQ4's `baseoq4/` runtime files.

## Experimental macOS Limitations

- macOS packages remain experimental Apple Silicon/arm64-only downloads. Intel Mac, universal2, and Rosetta release targets are not supported by this release line.
- The macOS Metal download is a Metal bridge around the OpenGL renderer, not a native Metal renderer. The legacy `platform_backend=native` Cocoa/OpenGL path is comparison-only diagnostic infrastructure, not a release backend.
- [GitHub issue #73](https://github.com/themuffinator/openQ4/issues/73) tracks launch-time `SIGSEGV` reports on some M4/M5 Apple Silicon systems in the Apple OpenGL 2.1/ARB2 startup corridor. Keep macOS support claims experimental until the issue is closed or the limitation is carried forward with matching signoff evidence in `docs/dev/macos-signoff-evidence.md`. If you hit this path, attach full terminal output as text, `openq4.log`, any matching `.ips` crash report, and the archive from `collect_macos_support_info.sh`; see [Experimental macOS Support Data](../../user/macos-support-data.md).
- Crash logs now include the `last renderer startup phase`, and normal logs include ordered renderer startup breadcrumbs through `first ARB2 interaction handoff` and `ARB2 interaction driver bypass` so issue #73 reports can identify the last ARB2/upload transition reached before a crash.
- Apple OpenGL 2.1 compatibility detection now uses normalized vendor/version and selected context facts for M4/M5/Tahoe-style reports, forces the CPU-backed vertex cache plus `SimpleInteraction.vfp`, bypasses ARB2 light interaction drawing on the unstable compatibility path, and fails closed with `Unsupported Apple OpenGL 2.1 compatibility path` if that required fallback shader pair cannot load. The bypass is a degraded-lighting fallback, not a full native macOS renderer fix.
- macOS packages include `SYMBOLS.txt`, and release assets include matching `openq4-<version>-macos-arm64-<bridge>-symbols.tar.xz` dSYM archives. Attach `SYMBOLS.txt` or the support collector archive with any `.ips` report so maintainers can choose the right symbols without asking you to rerun the crash.
- Moving only `openQ4.app` away from the package root now fails with a localized macOS startup error that names the adjacent-root contract, and support archives include `package/path-resolution.txt` with package/app paths plus copied `fs_basepath`, `fs_cdpath`, and `fs_savepath` log lines when available.
- Support archives now include `package/build-metadata.txt` with package/app `VERSION.txt` metadata, including openQ4 and `openQ4-game` commits when present, so maintainers can match a crash report to the exact engine and GameLibs source inputs.
- Support archives now include `system/rosetta.txt`, `logs/renderer-summary.txt`, `package/binary-architecture.txt`, `package/dylib-dependencies.txt`, `package/signing.txt`, and `package/quarantine.txt` so issue #73-style reports preserve translated-process state, renderer breadcrumbs, Mach-O architecture/dependency facts, `codesign`/Gatekeeper/stapler diagnostics, and quarantine presence without launching openQ4. The collector also copies recent app, loose-client, and loose-dedicated-server DiagnosticReports when macOS writes them, and it inspects any loose `x64` or `x86` executable names that are present without treating those paths as supported release targets.
- Experimental macOS packages remain on the Apple OpenAL framework. OpenAL Soft remains a migration-only build path for local Mac experiments until package layout, install names, signing, license, and notarization allowlists are deliberately added; support archives now include `logs/openal-summary.txt` with OpenAL vendor, renderer, device, and EFX lines copied from existing logs when available.
- Maintainers without Mac access can now run the `macos-static` validation profile to check static macOS policy, synthetic Apple GL 2.1 behavior, synthetic macOS package/archive fixtures, and release-matrix dry-runs without claiming macOS runtime coverage.

## Change Log

- Release workflow now builds a credential-aware package matrix, always keeps experimental macOS ARM64 OpenGL/Metal packages in the release, and verifies the expected signed DMG or unsigned archive names for the credential state that actually ran.
- macOS release packaging now keeps dSYM bundles out of runtime packages and publishes separate bridge-specific symbol archives with a runtime `SYMBOLS.txt` manifest for crash reports.
- macOS app bundles now include localized `OpenQ4PackageRoot.strings` resources, package validation requires them, and the support collector writes `package/path-resolution.txt` without launching the app.
- macOS support collection now writes `system/rosetta.txt`, `logs/renderer-summary.txt`, `package/binary-architecture.txt`, `package/dylib-dependencies.txt`, `package/signing.txt`, and `package/quarantine.txt` without launching the app, making Rosetta/translation, startup, architecture/dependency, Gatekeeper/stapling, quarantine, and accidental extra-architecture package reports easier to diagnose remotely.
- macOS OpenAL provider policy now keeps release builds on the Apple OpenAL framework, keeps `macos_openal_provider=system` migration-only for OpenAL Soft experiments, and adds `logs/openal-summary.txt` plus issue-template fields for OpenAL vendor, renderer, device, and EFX diagnostics.
- macOS native-backend containment now has static validation: Carbon and NSOpenGL stay confined to the comparison path, SDL3 Darwin sources and framework linkage stay Carbon-free, and native Metal remains blocked until a separate renderer design plan exists.
- Experimental macOS GameLibs alignment now has static validation across `openQ4` and `openQ4-game`: arm64 `.dylib` names, `@loader_path` install names, concrete macOS architecture build strings, package/support provenance for openQ4 and `openQ4-game` commits, and ARM64-sensitive allocation, savegame serialization, pointer-width, and alignment-sensitive allocation assumptions are guarded without requiring macOS runtime testing.
- The macOS no-runtime validation track is now explicit: `validate_macos_static.ps1`, `validate_macos_static.sh`, and `python tools/validation/openq4_validate.py macos-static` run static/policy tests, synthetic Apple GL 2.1 checks, synthetic macOS package/archive fixtures, shell syntax checks, and push/PR dry-runs, while optional renderer self-test binaries are limited to non-macOS hosts.
- Windows release builds now use the same optimized diagnostics profile as Linux and experimental macOS instead of debug binaries, with runtime asserts disabled for packaged performance.
- Windows video-memory detection now uses DXGI with a conservative fallback, so performance auto-detect no longer sees a bogus 100 TB VRAM value on fresh Windows runs.
- Runtime PK4 dependency tracking now follows generated source manifests instead of stale explicit file references.
- Runtime logs now begin with the openQ4 product name and version before the log-open timestamp.
- Air Defense 1 generated light-grid outputs were removed from source content without breaking local or CI packaging.
- Multiplayer clients now default to a spectator join state, use an intermission camera or spawn fallback for the initial view, and show a Q4-asset join screen with server info plus DM/team/tourney-appropriate join controls.
- Renderer hot paths cache stable counts and material state across GL, material, submit, and AAS loops.
- BSE culling, debris precache, and borderless-window defaults reduce outdoor frame pacing problems.
- System settings post-effects rows no longer overlap in compact display layouts, so Renderer Fallback and HDR Tonemap remain readable and selectable from the menu.
- Sound attenuation, pitch, diagnostics, reload behavior, and cinematic skip audio aging were aligned more closely with retail behavior.
- `testSound` is more useful for audio validation in scripted or unfocused launches: it can recover an allocated sound world for explicit diagnostic playback and pump the sound system immediately instead of only reporting that no world was active.
- `listSoundDecoders` now reports active OpenAL-backed channels from every allocated sound world and labels current versus inactive worlds, making fallback-world and scripted audio validation easier to read.
- Sound restarts and sample reload cleanup now tolerate sparse world, emitter, or channel lists while audio worlds are being rebuilt or torn down.
- Runtime `s_noSound` changes now detach active hardware voices, stop controller rumble, keep sound time fresh while audio rendering is disabled, and restart the sound backend when audio is re-enabled.
- Sound restarts no longer recreate queued-stream buffer pools while audio is disabled or after OpenAL fails to reinitialize, log when stream/sample reload work is skipped, and avoid a redundant second restart after manually re-enabling audio.
- Sound sample diagnostics, lookup, level-load cleanup, and sample-free voice cleanup now tolerate missing sample-table entries without silencing unrelated one-shot sounds.
- Game/menu sound-world switches now release the outgoing world's OpenAL voices immediately, so inactive worlds do not keep silent sources or occupy voice slots after a transition.
- Optional engine-side sound-world helper calls now no-op safely when the game or menu sound world is absent, while save/load sound-world operations still fail loudly instead of silently skipping data.
- Sound-world teardown, direct diagnostic playback, stop-all, reload, and pause/resume paths are more tolerant during map transitions and scripted audio probes.
- Sound emitters now clean up stale channel slots more consistently, so playback, stop/fade/modify commands, amplitude checks, and currently-playing queries are less fragile during audio reloads and map transitions.
- The sound-world mixer is more tolerant of sparse emitter/channel slots and cushion-fade cvar edge cases, improving stability in dense audio scenes and debug voice reporting.
- Sound save/load and demo cache handling now avoids malformed audio channel records and treats missing restored shaders as completed muted audio instead of invalid live channel state.
- Sound demo recording and playback now reject truncated fields, empty cached/start shader names, bad emitter indices, and unknown audio opcodes more clearly, tolerate null recording targets, write sound-start commands only after live channel creation succeeds, and allow valid local-emitter playback commands at index `0` without writing local-emitter allocation/free records.
- The post-June OpenAL voice, streaming, diagnostics, and source-lifecycle experiments have been removed from this release line. Audio behavior now matches the June 23 baseline while the newer work is reconsidered outside the shipped runtime.
- `applyPerformancePreset`, `autoDetectPerformancePreset`, the System-menu bridge, the settings registry, and `performancePresetSelfTest` now keep preset selection, auto-detection, command arguments, cvar effects, preset progression, localized choice labels, restart guidance, cvar state restoration, and release validation in sync.
