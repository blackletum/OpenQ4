# openQ4 0.8.1

## Highlights

- Native Wayland window and input startup is now guarded on both x64 and ARM64.
  The bundled SDL build includes optional libdecor plus native IBus/Fcitx
  composition for the stock Quake 4 character set, and repeated window,
  fullscreen, display, and mouse-capture transitions
  run under assetless Weston sessions before changes land.
- Linux clients no longer require PipeWire just to start. The bundled SDL
  audio backend loads PipeWire when it is available instead of making its
  runtime library a process-startup prerequisite.
- Native-Wayland stock single-player validation now covers a real save/restore
  cycle instead of stopping at map startup. The opt-in
  `tools/tests/linux_wayland_stock_sp_smoke.py` runner enters
  `game/airdefense1`, initializes OpenAL and the sound system, saves and reloads
  an isolated slot, captures active post-restore gameplay, and keeps the logs,
  save hashes, and screenshot needed to diagnose a failure. Automated audio
  evidence covers software initialization; audible playback remains an
  explicitly recorded human check.
- Linux ARM64 packaging now has native build/package coverage plus a real
  x64-hosted GNU/Linux AArch64 cross-build for the client, dedicated server,
  and SP/MP modules. The four-artifact gate verifies AArch64 ELF identity, the
  required PIE/RELRO/NOW/non-executable-stack properties, both `GetGameAPI`
  exports, and distinct SP/MP module bytes. Full-engine ARM64 portability is
  now checked with both compilers: Clang owns the native Wayland-only lane,
  while GCC continues to cover libdecor and dedicated-server builds.
- Linux dedicated servers now stay fully headless through startup, errors, and
  stock-map loading instead of opening splash, console, wipe, or loading-screen
  presentation paths. The x64 and ARM64 server executables also no longer need
  OpenAL, OpenGL/GLX, SDL, PipeWire, Wayland, or X11/Xext runtime libraries,
  and packaging now keeps both the server and its loaded MP module limited to
  core runtimes, making them suitable for minimal headless hosts. Native x64 validation hosts stock `mp/q4dm1`, accepts a
  separately packaged native-Wayland client, matches declaration checksums,
  reaches active gameplay, captures the client view, and shuts both processes
  down cleanly. Asset-free x64 and ARM64 CI keeps the module/startup gate covered;
  real ARM64 hardware stock-map signoff remains pending.
- Game-code portability can now be checked without a full engine build: the
  companion `openQ4-game` repository builds its distinct SP/MP modules with
  GCC or Clang on Linux x64 and ARM64 and validates their architecture,
  hardening, and required `GetGameAPI` export before integration.
- Repeated `vid_restart` no longer lets the game renderer tear down live Linux
  splash or system-console windows; each SDL video owner now has a matched lifetime.
- Experimental Apple Silicon macOS builds restore normal interaction lighting
  on the Apple OpenGL 2.1 compatibility path. Stock surfaces use a neutral
  GLSL lighting path, with a hardened simple-ARB fallback for surfaces that
  cannot use it, instead of defaulting the whole scene to flat ambient light.
- MSAA render-target failures now step down to a supported sample count rather
  than closing the game. If the driver rejects every offscreen target, openQ4
  keeps the session playable through direct rendering and records a detailed
  framebuffer diagnosis.
- macOS installs now use a self-contained `openQ4.app`: drag the app to
  `/Applications` without keeping a separate `baseoq4/` folder beside it.
  Runtime PK4 data is embedded as app resources, signed SP/MP modules use the
  standard Frameworks code location, and loose diagnostic/server binaries can
  still discover the sibling app runtime without duplicating the large payload.
- Linux and macOS system-console windows now scale correctly on HiDPI and Retina
  displays, keeping the output, input field, and controls readable across the
  full window instead of drawing them into its upper-left quarter.
- Linux downloads now include one-file AppImages for x86_64 and preview
  aarch64 alongside the existing archives. Each image comes from the same
  verified release payload and is launched through both Wayland and X11 in the
  release gate before publication.
- The runtime package no longer includes generated light-grid data for
  `game/airdefense1`, reducing package contents while preserving light-grid
  support for maps that still ship those baked outputs.

## Upgrade Notes

- Native Wayland remains the default in Wayland sessions. Set
  `OPENQ4_FORCE_X11=1` only when an XWayland fallback is needed; packagers can
  use `-Dlinux_x11=disabled` for a Wayland/EGL-only build. Explicitly enabling
  or disabling `linux_x11` forces the bundled SDL so that choice is honored;
  `auto` may use the system SDL and its compiled driver set.
- Source builders updating an existing checkout should purge the generated
  `sdl3` wrap extraction before rebuilding when files under
  `subprojects/packagefiles/sdl3/` changed. Meson does not reapply a changed
  patch directory to an already-extracted SDL tree, even with `setup --wipe`.
- Linux ARM64 downloads remain preview packages. They require 64-bit
  little-endian AArch64 hardware and a desktop OpenGL compatibility driver;
  GLES-only boards are not covered. Preview archive and debug-symbol filenames
  include `linux-arm64-preview` so their support tier stays visible after download.
  No reinstall or config reset is needed when
  the support tier changes, but first-class publication remains gated on
  accepted physical-hardware Wayland gameplay, server, audio, and input evidence.
- AppImages still require a legitimate Quake 4 installation for the retail
  assets. Run `chmod +x openq4-*.AppImage` once after downloading; use the
  `.tar.xz` package when you want the dedicated server, Steam Deck launcher, or
  loose offline documentation as ordinary files.
- Existing `r_appleARB2Interactions 0` settings automatically select the new
  lighting path; no config reset is required.
- `r_appleARB2Interactions 1` and `2` remain diagnostic simple/full ARB modes.
  Mode `3` is the emergency no-interaction fallback and produces deliberately
  degraded lighting. Run `vid_restart` after changing the mode.
- Existing extracted experimental packages can be replaced with the new
  `openQ4.app`; drag only the app to `/Applications`. Keep the full extracted
  folder only if you use the loose diagnostic client, dedicated server, or
  support collector. Do not add retail `q4base` files to the signed app.

## Change Log

- Linux ARM64 first-class publication now binds the exact physical stock-SP
  and dedicated/client JSON reports by SHA-256, validates their native ARM64
  Wayland and candidate-binary identity, requires the complete harness marker,
  screenshot/save, and declaration-checksum results, and rejects known
  VM/emulator evidence or undisclosed container use before promoting preview
  packages. Human audio, input, hotplug, and display checks remain separately
  reviewed rather than being inferred by automation.
- Alt-Tab and compositor focus changes now release fullscreen mouse capture,
  clear latched keyboard, controller, axis, and touch input, discard queued user
  input while inactive, and honor `s_muteUnfocused` on SDL3 Linux and macOS
  builds instead of leaving those behaviors Windows-only.
- Docking, unplugging a monitor, changing display scale or mode, and compositor
  hide/show transitions now refresh SDL3 desktop, framebuffer, cursor, and IME
  placement state without requiring a restart. Hidden windows release latched
  input and reacquire it only after the game window is visibly focused.
- Linux and macOS system-console input now decodes committed UTF-8 text before
  applying the stock single-byte font limit, so supported accented characters
  are entered once instead of as their raw UTF-8 bytes.
- OpenAL device hotplug notifications now use an atomic handoff from the audio
  callback instead of racing the update thread, improving recovery when playback
  devices appear, disappear, or the system default changes during play.
- Checksums and compiled declarations now stay architecture-stable on Linux
  x64 and ARM64: MD5, MD4, CRC-32, and Honeyman calculations use explicit
  32-bit words, while compiled tokens retain their 32-bit integer and signed-byte
  encoding. This prevents LP64 type widths from changing checksum or token data.
  A native-Wayland Linux x64 loopback run now loads stock `mp/q4dm1`, joins a
  second client with the same declaration checksum, spawns both players, reaches
  active gameplay, and captures both views before clean shutdown.
- Linux game modules now fail the build when they contain unresolved definitions
  and publish only the intended `GetGameAPI` entry point. This keeps GCC, Clang,
  x64, and ARM64 packages on the same loader contract instead of exposing
  compiler-specific helper symbols by accident.
- Linux validation now executes the native-Wayland window lifecycle with
  AddressSanitizer and UndefinedBehaviorSanitizer enabled. Engine/game code omits
  only the cross-module RTTI `vptr` check required by the runtime-loaded interface
  boundary; all other requested checks remain active and BSE keeps `vptr` coverage.
- macOS diagnostics now include a manually dispatched ASan+UBSan workflow for
  both OpenGL and Metal bridge builds. It instruments the client, dedicated
  server, and both game modules and provides a fail-fast assetless startup probe,
  helping platform-specific memory and undefined-behavior bugs surface earlier.
- macOS dedicated-server builds now run an asset-free hosted startup smoke for
  both package bridge configurations. The gate loads the staged multiplayer
  module, reaches an initialized server frame, and requires clean shutdown;
  stock-map and physical-hardware multiplayer signoff are still required.
- Developers now get experimental Intel x64 macOS CI for both OpenGL and Metal
  bridge configurations, including engine and game-module architecture checks,
  renderer startup, and dedicated-server module loading. This release still
  publishes Apple Silicon packages only; Intel and universal2 downloads await
  real Intel gameplay, packaging, signing, and Gatekeeper evidence.
- Universal2 runtime groundwork lets future merged engine slices share a
  trusted merged SP/MP module pair while preserving current thin-module lookup.
  Universal packages are not published yet; merge, symbol, signing,
  notarization, and dual-architecture runtime gates remain mandatory.
- A hosted universal2 assembly gate now combines matched Apple Silicon and
  Intel thin inputs for both graphics bridges, verifies every merged code item
  contains exactly both native slices, packages and symbol-checks the app, and
  performs an assetless dedicated-server startup check. Downloads remain
  Apple-Silicon-only until final hardware and release-signing evidence exists.
- A manual non-publishing universal2 release-candidate workflow now produces
  retained evidence artifacts for both bridge variants and can re-sign,
  notarize, staple, and assess the final merged app with Developer ID
  credentials. User-facing macOS downloads remain experimental and arm64-only
  until hosted and real-hardware signoff is complete.
- Script-register string values containing `%` are now returned unchanged in
  both single-player and multiplayer instead of being treated as format
  directives during script/editor inspection.
- Hardened several rare game-code paths that modern Linux diagnostics exposed:
  multiplayer voice routing no longer writes one slot past its cache, AI vehicle
  ejection uses a valid full-range command, and tactical/animation failure paths
  clean up or stop safely.
- Long session commands and admin-ban GUIDs are now always terminated; empty or
  quoted text and high-byte filenames no longer trigger invalid memory access;
  malformed legacy model data and empty silhouettes fail cleanly; and SP/MP AI
  obstacle path refinement now compares the intended navigation areas.
- Direct build-directory launches now include the required openQ4 runtime
  manifest alongside their generated PK4s and game modules. Early Linux/macOS
  startup errors also retain their original fatal diagnostic instead of touching
  released console-setting storage during shutdown.
- Generated Air Defense 1 light-grid metadata and per-area atlas files are no
  longer included in the runtime package; the map continues to load using its
  baseline lighting path.
- Renderer frame memory and its backing blocks are now explicitly 16-byte aligned,
  preventing invalid command and temporary-data access across Linux x64 and ARM64
  allocator configurations.
- Legacy OpenGL vertex-cache draws now handle a valid VBO at byte offset zero
  without treating it as a null object, improving renderer stability across
  ambient, shadow, material, visibility, and diagnostic passes.
- Empty and duplicate-free map surfaces now skip invalid zero-length memory
  operations, improving stock map loading under strict Linux runtimes.
- Stock LWO model loading now initializes vector hash grids deterministically,
  avoiding undefined integer overflow from an uninitialized hash size.
- SP and MP entity and client-entity references now preserve legacy spawn IDs
  through defined unsigned bit packing, preventing stock entity construction
  from tripping a signed-shift failure on Linux x64 or ARM64.
- Animated-model joint handles now use their intended 32-bit integer-index
  representation, allowing stock model definitions to load on strict Linux
  runtimes without rejecting valid nonnegative joints.
- Multiplayer mute/friend controls now reject invalid client selections, and
  debug-HUD masks clamp invalid indices, avoiding unsafe bit shifts from empty
  stat-window selections or out-of-range console values.
- State-thread stage and delay results now retain stock mover payloads as their
  intended integer protocol, avoiding strict-runtime failures on encoded delay
  values while preserving save compatibility.
- Empty animation metadata and other zero-length ranges now load safely through
  the shared SIMD copy/set/zero path, preventing strict Linux runtimes from rejecting
  valid null-backed empty containers during stock SP/MP map startup.
- Stock AI animation selection now keeps its established random sequence through
  defined 32-bit wraparound, preventing strict Linux runtimes from stopping on
  signed overflow while retaining save and network seed compatibility.
- Player inventory restore and weapon-mod grants now handle the normal
  no-current-weapon state safely, preventing an out-of-bounds access during stock
  SP/MP spawn.
- First-person weapon setup now starts from defined light and flashlight state,
  preventing strict Linux runtimes from stopping during stock player spawn while
  preserving the intended muzzle-flash shadow-suppression fallback.
- Multiplayer client 0 now receives a real muzzle-flash light ID instead of the
  renderer's unset sentinel, keeping first-person player and weapon shadows hidden
  under weapon lights consistently with single-player.
- Player focus detection now starts from a defined neutral state, avoiding a
  strict-runtime failure on the first active SP or MP gameplay frame.
- The first rendered gameplay frame now accepts an initially empty draw-surface
  list safely, preventing a strict Linux runtime stop before normal scene drawing.
- Repair-bot effects and first-frame nailgun target locking now start from defined
  timers and states, avoiding platform-dependent behavior in those stock gameplay
  paths.
- Saves made immediately after entering a map now contain initialized tactical-AI
  sensor state, preventing strict Linux runtimes from stopping before an AI has
  performed its first tactical search.
- Particle-physics objects now start with a defined unbound orientation state,
  allowing immediate stock-map saves to serialize items consistently on strict
  Linux runtimes.
- Rigid-body physics now starts with its drop-to-floor solid-test state disabled
  explicitly, preventing immediate stock-map saves from reading an indeterminate
  flag on strict Linux runtimes.
- Immediate saves now preserve deterministic actor death-push, Strogg Marine
  attack, and Harvester turn/sweep state, while the Makron's bolt-sweep
  completion state starts disabled until combat initializes it.
- Vehicle positions now initialize their previous command, view-angle, and
  impulse-sequence history, making first-frame input comparisons and immediate
  save data deterministic across Linux architectures.
- Vehicle weapons now initialize their animation and muzzle-flash timing before
  first use, making immediate saves deterministic and applying the authored
  flash duration consistently from the first shot.
- Single-player save/reload and map transitions now release PVS arrays with the
  matching allocator operation, preventing strict Linux runtimes from aborting
  after a save has completed.
- Harvester projectiles and gauntlet/dark-matter effects now restore through
  alignment-safe, type-checked object pointers, preventing 64-bit save loads
  from overwriting their compact entity-reference state.
- Restored players now resume with a defined neutral emote state, preventing
  strict Linux runtimes from rejecting the first post-load idle animation frame.
- Dedicated multiplayer servers now handle an empty repeater/viewer allocation
  during startup, shutdown, and capacity changes, avoiding strict-runtime aborts
  before the stock map can open its socket or while repeater storage is resized.
- Empty multiplayer rankings and terminal tram, spline, and vehicle paths now
  skip zero-item sorts, improving strict-runtime stability during server
  startup/disconnect and on stock transport routes with no next target.
- Map-compilation cleanup now handles long map paths without overflowing its
  region/leak filename buffer, and malformed ARB program targets fail safely.
- Fixed generated AAS area output to write the exact field count consumed by the
  loader, and made cvar flag listings display `%` characters literally instead
  of treating names or descriptions as format directives.
- Screenshots on native Wayland now capture the finished scene reliably instead
  of showing colored noise or black bands after the window is presented.
- Stock maps and mods load substantially faster from case-sensitive filesystems:
  exact-case loose-file probes no longer rescan every parent directory before
  falling back to mixed-case recovery.
- Kept distant sky and sun shadows readable by automatically tightening their
  filtering relative to local projected lights, including the optional soft-
  shadow path. The new `r_shadowMapDistantFilterScale` setting can restore the
  former shared softness (`1`) or sharpen distant shadows further (`0`).
- Added symbolic FBO status, OpenGL context, attachment format/size/sample, and
  requested/effective MSAA logging for renderer support reports.
- Hardened CPU-backed vertex/index-array state and generated-character
  interaction fallback on Apple GL 2.1.
- Removed repeated `OSPathToRelativePath failed` warnings for already-relative
  game script include paths.
- Added required hosted macOS assetless launch smoke for both OpenGL and Metal
  bridge builds, plus a packaged-app path-resolution smoke from an unrelated
  working directory.
- Release packages now come from one reproducible source pair across every
  platform: the release run pins the triggering openQ4 commit and resolves the
  selected `openQ4-game` revision once to a full commit SHA. Packaging stops if
  a checkout, staged source manifest, existing version tag, or published release
  target points at anything else.
- Linux runtime and debug-symbol downloads now receive another integrity check
  after symbols are separated and after the runtime archive is created. The
  upload is rejected if stripping changes the ELF architecture, hardening,
  module ABI, or dependencies, or if a packaged binary no longer matches its
  detached debug file and build ID.
- Linux ARM64 first-class releases now require signoff for the exact bytes being
  promoted. A non-publishing ARM64-only candidate run supplies the exact archive
  and a structured evidence record. Placeholder attestations and invalid review
  dates are rejected, the final ARM64 build rejects a different source pair,
  version, package name, archive hash, or client/server/SP/MP binary hash, and
  the publishing job rechecks the downloaded archive while allowing only the
  expected release filenames. Preview downloads and their support wording
  remain unchanged.
- Linux release clients now prove both native Wayland and X11/XWayland startup
  before packaging. The exact stripped x64 and ARM64 staged clients run bounded,
  assetless renderer/window lifecycle checks under headless Weston and Xvfb,
  with driver/backend and clean-teardown markers required and failure logs
  retained for diagnosis. This does not change the ARM64 preview support tier.
- Expanded the packaged support collector's renderer summary with interaction,
  render-target/MSAA, filesystem-root, and selected game-module diagnostics.
- Missing-game-module reports now show the trusted package roots and exact
  module paths that were checked, making damaged or older macOS package layouts
  easier to diagnose.
- If macOS finds a game module but the platform loader rejects it, the log and
  support archive now retain the selected module plus the `dlopen` reason.
- Startup guidance for older adjacent macOS packages now clearly distinguishes
  them from current self-contained apps, which can be dragged to `/Applications`.
- macOS support archives now include a privacy-filtered renderer-settings
  summary, making graphics-setting regressions easier to diagnose without
  collecting bindings, account/profile, network, audio-device, or unrelated
  configuration.

## Known Limitations

- Linux ARM64 packages remain preview. Hosted checks prove the four release binaries and assetless
  Weston/Xvfb window, compositor, and input startup only. Real ARM64 hardware
  running native Wayland still needs SP and MP gameplay with stock assets plus
  stock-map dedicated-server, audio/input, display, and package signoff; GLES-only
  boards are not covered by the desktop OpenGL package.
- Native IME composition and candidate windows are supported, but stock Quake 4
  edit widgets and bitmap fonts remain single-byte; unsupported committed
  characters are ignored instead of being displayed as corrupt text.
- This remains experimental macOS support for Apple Silicon/arm64 only. Intel Mac,
  universal2, and Rosetta packages are not supported or published.
- The `-metal` package is a Metal bridge around the OpenGL renderer, not a native Metal renderer.
- `platform_backend=native` remains comparison-only diagnostic infrastructure, not a release backend.
- [GitHub issue #73](https://github.com/themuffinator/openQ4/issues/73) remains
  open until the new default is verified in SP and MP on real Apple hardware.
  Accepted signoff is still required on the documented macOS 11 floor and the
  latest public macOS in `docs/dev/macos-signoff-evidence.md` before support can
  be promoted beyond experimental.
- Unsigned archive builds are ad-hoc signed but are not Developer ID notarized;
  macOS may require explicit approval. See
  [Experimental macOS Support Data](../../user/macos-support-data.md) when
  reporting a failure.
