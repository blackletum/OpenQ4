# openQ4 0.6.5 Release Notes

## Highlights

- Runtime packaging is more robust: openQ4 PK4s now track `content/baseoq4/pak0/` and `content/baseoq4/pak1/` through generated source manifests, so removed generated content no longer leaves stale build dependencies.
- Windows and Linux release packaging is unblocked when Apple signing credentials are not configured. The release workflow keeps Windows/Linux artifacts mandatory and includes signed/notarized macOS DMGs only when the Apple Developer ID and notary secrets are present.
- The renderer does less repeated CPU work in GL, material, submit, and AAS-heavy paths by caching stable loop counts, stage metadata, material masks, and texture binding state.
- Windows windowed play defaults to a smoother borderless presentation path, reducing OpenGL swap stalls seen in BSE-heavy outdoor scenes such as Air Defense 1.
- Sound playback is closer to retail Quake 4, including attenuation, master volume, frequency-shift behavior, restart/reload diagnostics, and cinematic skip timing.
- macOS push verification builds are healthy again after replacing raw C string compares in process handoff and environment filtering with openQ4's guarded string comparison API.

## Upgrade Notes

- This release publishes Windows x64, Windows ARM64, Linux x64, and Linux ARM64 packages, including Windows installers and Linux debug-symbol archives.
- The `v0.6.5` release does not include macOS downloads because the repository does not currently have Apple signing and notarization secrets configured. The macOS DMG lane remains available for credentialed release runs and will not publish unsigned or ad-hoc-signed official macOS artifacts.
- Replace an existing openQ4 package with the matching new archive or installer, keeping your retail Quake 4 `q4base/` assets separate from openQ4's `baseoq4/` runtime files.

## Change Log

- Release workflow now builds a credential-aware package matrix and verifies the artifact list against the platforms that actually ran.
- Runtime PK4 dependency tracking now follows generated source manifests instead of stale explicit file references.
- Air Defense 1 generated light-grid outputs were removed from source content without breaking local or CI packaging.
- Renderer hot paths cache stable counts and material state across GL, material, submit, and AAS loops.
- BSE culling, debris precache, and borderless-window defaults reduce outdoor frame pacing problems.
- Sound attenuation, pitch, diagnostics, reload behavior, and cinematic skip audio aging were aligned more closely with retail behavior.
