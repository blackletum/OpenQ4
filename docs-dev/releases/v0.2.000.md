# openQ4 v0.2.000

## Highlights

- Fixes the initial Windows `v0.2.000` package startup failure where the client could show the splash screen and exit before reaching the menu.
- Physical/DVD-style Quake 4 installs that include the core retail media set now pass startup validation without needing Steam-added patch/menu PK4s such as `pak023.pk4` through `pak025.pk4`.
- The packaged `baseoq4` manifest is now generated from the actual engine release version, so release archives and installers stay loadable when the workflow bumps versions.
- `requiredOpenQ4Version` is treated as a minimum compatible engine version, while the manifest's own `version` remains the packaged content version.
- Stable release packaging now fails early if `baseoq4/mod.json` has a stale content version or requires a newer engine than the package contains.

## Upgrade Notes

- Replace any earlier `v0.2.000` Windows package or installer that closes immediately after the splash screen.
- You do not need to fetch optional Steam patch/menu PK4s if your legitimate Quake 4 install already has `pak001.pk4` through `pak022.pk4`.
- No save-game, config, or asset changes are required.

## Change Log

- Corrected release staging so `baseoq4/mod.json` uses the effective Meson `version_base_override` value.
- Limited required official media validation to the core retail PK4 baseline and documented optional official patch/menu and language packs.
- Added a packaging validation guard for stable releases to catch stale manifest versions and impossible `requiredOpenQ4Version` requirements before artifacts are published.
