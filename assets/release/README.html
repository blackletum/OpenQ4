openQ4

Release package for the open-source Quake 4 engine and game-code replacement.

This archive contains openQ4 binaries plus the staged baseoq4/ runtime payload. Quake 4 assets are not included.

Open Documentation Portal Project Overview Technical Reference

Requirements

  • A legitimate Quake 4 installation from Steam or GOG.
  • The stock retail PK4 files in a q4base/ directory, available through openQ4's asset discovery or a manual fs_basepath setting.

Getting Started

  1. Install Quake 4 from Steam or GOG and leave its retail q4base/ folder intact.
  2. Install or extract openQ4 to its own folder when possible. On Windows, either run the installer executable or extract the archive; the installer can detect an existing openQ4 install, adds a standard uninstall entry, and optionally registers openq4:// browser links.
  3. Use a clean layout: preferably keep retail Quake 4 separate with assets under q4base/; for a portable setup, keep a copied retail q4base/ beside openQ4's baseoq4/. Never copy retail PK4s into baseoq4/.
  4. Run openQ4-client_<arch> or openQ4-client_<arch>.exe.
  5. On Steam Deck, use openQ4-steamdeck when it is included in the package. Direct openQ4-client_<arch> launches on Steam Deck or SteamOS also auto-select the Deck profile when com_platformProfile is still default.
  6. Open docs/index.html for the full offline documentation set, or use the installer-created Start Menu shortcuts on Windows.
  7. If auto-detection fails, launch with +set fs_basepath "path\\to\\Quake 4", pointing at the retail root that contains q4base/.

Steam Deck Notes

  • The Deck launcher enables the steamdeck platform profile and sets OPENQ4_STEAMDECK=1.
  • Native Wayland is the default SDL path when available. Set OPENQ4_FORCE_X11=1 to force the XWayland fallback from the Deck launcher or a direct client launch.
  • If native Wayland hits a compositor/libdecor startup or decoration issue, set OPENQ4_WAYLAND_DISABLE_LIBDECOR=1 for that launch.
  • Deck controller tuning lives under Settings -> Game Options -> Controller, including gyro, touchpad mode, touchscreen routing, and low-battery rumble caps.
  • Run listControllers from the console when reporting Deck input, battery, gyro, touchpad, or touchscreen issues.

macOS Notes

  • macOS packages are for Apple Silicon/arm64 Macs. Intel Mac and universal2 packages are not published yet.
  • Credentialed macOS release runs publish signed/notarized OpenGL and Metal bridge DMGs. Releases without Apple Developer ID signing and notarization credentials publish clearly labeled -unsigned.tar.gz archives instead.
  • Unsigned macOS archives are ad-hoc signed only for bundle validity, are not notarized, and may require normal Gatekeeper approval on first launch.
  • The Metal package is a bridge mode around the existing OpenGL renderer, preserving stock-asset compatibility while Metal translation-layer work continues.
  • Open the DMG or unpack the -unsigned.tar.gz archive, then launch openQ4.app. Keep openQ4.app, baseoq4/, and the loose runtime files together.

Package Contents

  • openQ4-client_<arch> and openQ4-ded_<arch>: engine binaries.
  • *.pdb files on Windows: diagnostic symbols that make crash dumps actionable.
  • baseoq4/pak0.pk4: openQ4-owned runtime content and overrides.
  • docs/index.html: full packaged HTML documentation portal.
  • docs/docs-user/ and docs/docs-dev/: converted user and developer references.
  • VERSION.txt: packaged version metadata.
  • LICENSE: GPLv3 engine license text.

Crash Reports

On Windows, unhandled openQ4 crashes write a crash log and memory dump under crashes/ beside the executable. Attach both generated files when reporting a crash.

Documentation

The release archive includes a full offline HTML docs site instead of only a single readme. Start with docs/index.html, then move into the overview, build guide, technical reference, user guides, and developer notes.
